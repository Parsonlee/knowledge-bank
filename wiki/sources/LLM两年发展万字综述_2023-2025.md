# 万字综述，讲一讲这两年大模型这整个领域到底发展了哪些方面

## 核心要点

### 一、2023年 GPT-4 时代——唯规模论终结

- GPT-4 是 Scaling Law 顶峰体现：Transformer 架构 + 更大规模；上下文从 4K/2K 扩展到 8K/32K
- GPT-4 以保密架构为由（CloseAI）拒绝公开细节，强化了"Scaling is all you need"信念
- **裂痕**：传统 Dense Transformer 的 O(L²) 注意力复杂度 + 高推理成本 = 限制扩展；纯规模不赋予多步推理能力

### 二、2024年——效率的迫切需求

#### 2.1 MoE 稀疏化
- **核心思想**：用大量小型专家网络替换 Dense FFN，门控路由器（router）动态选择少数专家
- DeepSeek-V2：236B总参数，每token激活21B（>10:1比）；DeepSeek-R1：671B总参，37B激活
- Qwen3：同时提供 Dense 模型（最高32B）和 MoE 模型（30B-A3B, 235B-A22B）双轨策略
- Minimax-m1：456B总参，45.9B激活，32个专家，约10:1已成行业基准

#### 2.2 注意力机制革命
- **DeepSeek MLA**（Multi-Head Latent Attention）：K/V 压缩为低秩潜在向量，支持128K上下文同时KV Cache减少93.3%；V3进一步动态低秩投影+分层自适应缓存
- **Minimax-m1 Lightning Attention**：线性复杂度O(L)；每7个线性注意力块插入1个完整softmax注意力块；100K生成FLOPs仅为R1的25%，支持100万token上下文
- **Qwen2.5 GQA**：分组查询注意力，现代LLM标准优化

#### 2.3 效率作为战略护城河
- 开源（DeepSeek/Qwen）：公开 MoE/MLA 细节，以低成本接近 SOTA，冲击高成本壁垒
- 闭源（OpenAI/Anthropic）：同样投入效率，但将节省的算力用于专有能力（extended thinking/agentic tool use），护城河在"效率解锁的独特能力"

### 三、2025年——推理 Thinking 走向台前

#### 3.1 CoT/Thinking 模型
- **OpenAI o系列**：回答前生成隐藏思维链；o1 解决 AIME 83% vs GPT-4o 仅13%；推理时计算成为新 Scaling Law
- **Claude 3.7/4**：首个"混合推理模型"（hybrid reasoning model），支持 fast/extended thinking 用户选择
- **Gemini 2.5 Pro**：屠榜
- **Qwen3**：Thinking/非Thinking 模式 + thinking budget 参数

#### 3.2 RL 为推理而训练
- RL 从对话对齐工具演进为教授模型推理的核心方法
- DeepSeek-R1：冷启动SFT → 大规模RL（GRPO算法，规则验证奖励）→ SFT → 全场景RL；自发涌现自我验证/错误修正（Aha moment）
- Minimax-m1 CISPO：GRPO变体，裁剪重要性采样权重，456B模型3周完成RL训练

#### 3.3 智能体工具使用
- o3/o4-mini：首批"智能体工具使用"模型，自主决定组合使用网页搜索/Python/DALL-E
- Claude 4：代码执行Sandbox + Files API + MCP工具 + computer use（鼠标键盘操作）

### 四、竞争格局与未来

- **Benchmark 分化**：MMLU等饱和 → AIME/GPQA/SWE-bench成新标准
- **SOTA不再单一**：Claude 4代码SOTA、o3/o4-mini数学SOTA、Gemini 2.5上下文/多模态SOTA
- **后续方向**：具身智能（Embodied AI）+ 世界模型；后Transformer架构探索（SSM等改进而非取代）

### 五、现代AI架构三大支柱总结
1. **效率（Efficiency）**：MoE稀疏化 + 先进注意力机制（MLA/混合注意力）
2. **推理（Reasoning）**：推理时算力重分配 + RL训练涌现解题能力
3. **智能体（Agency）**：推理能力的应用，自主工具调用与数字/物理世界交互
