# 北大RAG综述_AIGC检索增强

## 摘要

北京大学崔斌教授 PKU-DAIR 实验室系统调研 RAG（Retrieval-Augmented Generation for AI-Generated Content），覆盖近 300 篇论文。综述按组件拆解 RAG 系统：检索器与生成器的交互方式、提升方法、多模态应用、评估标准。

## RAG 基础方法（四类）

按检索器如何与生成器交互分类：

### 1. 基于查询的方法（Query-based）
- 检索内容与原始查询合并为 Prompt（提示扩充）
- 最广泛采用，可直接结合提示工程、少样本学习
- 适用于基于 LLM 的文本生成任务

### 2. 基于隐空间表达（Latent Representation）
- 检索内容以 latent representation 在模型内部与生成过程交互
- 边生成边交互，包括拼接或注意力机制融合
- 提升生成模型理解能力

### 3. 基于概率表示（Logit-based）
- 解码阶段利用概率方法融合检索信息
- 对检索结果概率分布求和或融合，逐步计算生成概率
- 权衡不同检索信息的相关性和可信度

### 4. 投机 RAG（Speculative RAG）
- 用检索替代部分/全部生成过程
- 检索成本低于生成成本时有应用潜力
- 如投机推理：检索替代小模型生成，大模型验证

## RAG 提升方法（五类）

### 1. 输入提升
- **查询改写（Query Transformation）**：修改输入查询提高检索结果
- **数据增强（Data Augmentation）**：检索前改进数据（去无关/消歧义/更新/合成）

### 2. 检索器提升
- **递归检索**：拆分查询多次搜索
- **块优化**：调整 chunk size
- **检索器微调**：微调嵌入模型
- **混合检索**：统计词频 + 向量相似度
- **重排序**：对检索内容重新排序
- **元数据过滤**：利用时间/目的等元数据过滤

### 3. 生成器提升
- **提示词工程**：压缩/回退/主动/思维链提示
- **解码过程控制**：超参数调整/输出约束
- **生成器微调**：领域知识或匹配检索器

### 4. 结果提升
- **结果改写**：另一模型编辑/改写生成结果

### 5. RAG 流程提升
- **自适应检索（[[概念_Adaptive-RAG|Adaptive-RAG]]）**：判断是否需要检索（规则/模型判断）
- **迭代 RAG**：迭代检索+生成，反映缺乏知识并补充

## 评估标准

- **忠实性**：从检索内容推断正确答案的能力
- **答案相关性**：生成结果是否解决问题
- **上下文相关性**：检索内容是否包含足够知识
- 其他：噪声稳健性、拒绝回答能力、信息整合能力、反事实稳健性

## 局限性

- 检索结果噪声（ANN 近似匹配、信息编码损失）
- 额外开销（延迟、存储复杂度）
- 检索与生成复杂交互（目标不同、潜在空间不一致）
- 长上下文生成挑战（上下文窗口限制）

## 未来方向

- 更先进的基础方法与提升方法
- 更高效的 RAG 部署（不仅 LLM 文本，还有多模态）
- 发挥长尾和实时知识能力（动态更新知识库）
- 与微调/强化学习/Agent 等技术结合
