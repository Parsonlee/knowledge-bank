## 核心要点

### 两大分类框架

- **提示工程**：检索增强（RAG）/ 通过反馈自我完善 / 提示微调
- **模型开发**：新的解码策略 / 利用知识图谱 / 基于忠诚度的损失函数 / 有监督微调

### 提示工程——RAG（按时间分类）

- **生成之前**：LLM-Augmenter（先检索证据链再提问，迭代验证）；FreshPrompt（搜索引擎整合最新信息进提示，适合动态场景）
- **生成过程中**：知识检索（logit 识别潜在幻觉并纠正）；D&Q 框架（监督微调阶段+预测阶段外部工具查询）；EVER 框架（实时逐步三阶段：生成→验证→纠正）
- **生成之后**：RARR（自动归因，研究+后期编辑对齐证据）；高熵词汇检测替换（替换高熵词降低幻觉，处理缩略语歧义）
- **端到端**：Seq2Seq + DPR 密集向量索引集成，生成时以输入和检索文档为条件

### 提示工程——反馈自我完善

- 可靠性分解（普适性/社会偏见/校准/事实性四方面）
- ChatProtect（检测并处理自相矛盾幻觉）
- 自我反思方法（医疗 QA，迭代反馈改善事实性/一致性/蕴含）
- SC 结构化比较推理（提出比较方面+成对一致性比较器）
- Mind's Mirror（将 LLM 自我评估能力蒸馏进 SLM）
- DRESS（条件 RL 处理视觉语言模型对齐）
- MixAlign（不确定时生成问题求用户澄清）
- **Chain-of-Verification（CoVe）**：初始答复→验证问题→独立回答→最终精练输出
- CoNLI（自然语言推理链幻觉检测 + 后期编辑，无需模型调整）

### 提示工程——提示微调

- UPRISE（零样本任务自动检索提示，可跨任务泛化）
- SynTra（合成任务上微调，能力迁移至摘要任务）

### 模型开发——解码策略

- **CAD（上下文感知解码）**：对比有/无上下文输出分布差异，上下文与先验冲突时效果更好，无需额外训练
- **DoLa（对比层解码）**：对比前后层 logit 差异，利用特定 Transformer 层中事实知识局部化
- **ITI（推理时介入）**：识别高探测准确性的稀疏注意力头，推理时沿真实性方向移动激活，显著提升 LLaMA TruthfulQA 性能

### 模型开发——知识图谱

- RHO（KG 实体与关系谓词表示，局部+全局知识基础技术）
- FLEEK（无模型偏置的事实核查工具，查询知识图谱和开放网络，可解释）

### 模型开发——损失函数

- THAM（信息论正则化，文本幻觉正则化损失，减少不加区分复制）
- mFACT（跨语言摘要忠实度，按忠实度分数加权损失）

### 模型开发——有监督微调

- 知识注入+师生模型（用 GPT-4 引导弱 LLM，领域微调注入知识）
- 幻觉增强复述（利用 LLM 幻觉创建对照数据集，增强模型归属）
- 事实性微调（自动事实检查 + 偏好学习，Llama-2 无需人工标注显著降低错误）
- BEINFO（行为微调增加信息，在含真实知识来源对话上微调）
- R-Tuning（拒绝意识指导调节，训练 LLM 对超出能力的问题保持沉默）
- TWEAK（假设验证模型 HVM 对候选排名，仅调整解码过程无需重训生成模型）
