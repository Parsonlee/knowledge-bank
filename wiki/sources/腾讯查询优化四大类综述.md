# 腾讯查询优化四大类综述

## 摘要

腾讯最新综述 "A Survey of Query Optimization in Large Language Models"（arXiv:2412.17558），系统分类 LLM 中查询优化（QO）技术为四大类：查询扩展、问题分解、查询消歧、查询抽象。查询优化对 RAG 效率和质量至关重要。

## 四大类查询优化

### 1. Query Expansion（查询扩展）

#### 内部扩展（Internal Expansion）
- 针对信息时效性较低的查询，利用 LLM 预训练知识
- 技术：LLM 生成上下文文档 / 少样本伪文档 / 零样本假设文档([[概念_HyDE|HyDE]]) / 多样查询生成相互验证 / 关键词释义
- 适用：信息已包含在 LLM 预训练阶段的查询

#### 外部扩展（External Expansion）
- 针对高度时效性查询，需从知识库或网络搜索事实
- 技术：LLM 结合潜在答案扩充查询 / 初始检索文档关键信息优化查询 / 挖掘语料库知识评估相关性 / LLM 生成伪参考增强检索器
- 适用：需要最新事实的高时效性查询

### 2. Question Decomposition（问题分解）

- 将复杂查询分解为更简单可回答的子查询，分别检索后整合响应
- 技术：少样本提示分解 / 制定计划划分子任务 / 强化学习迭代改进 / 多跳查询分解+CoT / 子方面探索器剖析查询
- 适用：需要检索多个事实形成全面答案的复杂查询

### 3. Query Disambiguation（查询消歧）

- 识别并消除复杂查询中的歧义，确保单一精确解释
- 技术：自然语言演绎推理格式 / 重述查询再推理 / 构建消歧树 / rewrite-then-edit 框架 / 对话答案建模优化改写器 / 多方面反馈探索最优改写策略
- 适用：模糊查询，可能有多种解释

### 4. Query Abstraction（查询抽象）

- 提炼查询核心概念创建高层次表示，获取更广泛视角
- 技术：精心设计提示引导 LLM 推理 / 概念推理在可验证符号空间生成解决方案 / 抽象变量推理链+领域工具 / 生成高层抽象信息作查询背景([[概念_Step-back提示|Step-back]])
- 适用：需要理解领域特定推理及数据上下文的查询
