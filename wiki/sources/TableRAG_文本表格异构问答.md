# TableRAG_文本表格异构问答

## 来源信息

- **标题**：TableRAG_文本表格异构问答
- **作者**：华为云信息检索团队
- **发布时间**：2025-06-18
- **URL**：https://mp.weixin.qq.com/s/synQYJYw6B8b8QAkofitaw


> 华为云提出的 TableRAG 混合推理框架，融合文本检索与 SQL 数据库操作，解决文本+表格异构文档问答难题。

## 问题背景

传统 RAG 处理含表格文档的缺陷：
1. 结构化信息严重丢失：表格序列化为文本后行列结构被"压扁"，关系线索丢失
2. 缺乏全局推理视角：文档切割为局部片段后，难以应对需要整合全局信息的多跳推理

## TableRAG 框架

### 离线阶段（知识库构建）

- 文本知识库：文档文本 + 表格 Markdown 序列化 → 切分为 chunks → BGE 模型向量化
- 表格数据库：
  - Schema 提取 & 建表：提取表名/列名/列类型
  - 建立映射关系：每个含表格的 chunk 与其所属"表格 schema"建立映射
  - 数据库接入：表格数据导入关系型数据库（如 MySQL）

### 在线阶段（迭代式推理）

四个关键步骤：
1. **上下文感知的查询拆解**：结合数据结构特征判断子问题应查表还是查文档
2. **文本检索**：向量召回 + 重排序，挑出 top-k 相关内容
3. **基于 SQL 的表格查询**：NL2SQL（大模型驱动）生成 SQL 查询语句，直接执行
4. **中间答案组合**：对比 SQL 结果和文本片段一致性，动态调整权重得出最终答案

### 迭代终止

系统搜集足够信息后迭代结束，输出整合后的最终答案。大多数问题在 5 步内解决（约 63%）。

## HeteQA 评测集

- 304 个高质量问题，涵盖 9 大领域、5 类核心表格推理方式
- 136 张维基表格，5314 个知识实体
- 构建方法：Claude 3.7 辅助人工标注（自动生成问题 → 人工校验答案 → 添加文本指代）

## 实验结果

- TableRAG 比现有最强方法提升至少 10%
- 对不同 LLMs 适配性强，通用性好
- 错误率最低：得益于上下文感知拆解 + 合适的数据库操作策略

## 局限性

- 复杂表格结构（层级表头、单元格合并）兼容性
- 多表联合查询和嵌套查询准确性待提升
- 未涉及模型微调
