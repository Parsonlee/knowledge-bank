## 核心要点

### 复现 R1 的四大关键

- **base model and data distribution**：RL 是发掘模型潜力的方法，模型没有潜力则 RL 无效；数据质量同样关键，需统计多维指标（每条 prompt 的采样准确率、输出长度、是否能直接答出、ground truth 是否易验证），考虑用课程学习动态调控数据
- **scaling is all you need**：R1 的成功不是宣判 PRM 死刑，而是说明 100W 条 ORM 数据 > 1W 条 PRM 数据；深度学习范式 = "scaling + 雕花"，ORM 属于 scaling，PRM 属于雕花，应先跑通 ORM 再优化 PRM
- **infra is all you need**：长 CoT 导致单条回复上万 token，显存压力和解码耗时骤增；vLLM/sGLang 已成 RLHF 框架标配；GRPO 省掉 critic_model 的关键价值是降低训练框架维护难度；各公司复现进度不一，大概率是 RL infra 水平参差不齐
- **hyper-parameter is all you need**：学习率、KL 系数、回报折扣系数、n_samples、RL 算法都需要认真调；没有技术自信和算力支撑，不会花时间大规模调参

### 四大常见误区

- **误区1**：r1 的关键在于对 base 模型做 RL ——实际上 r1 是基于 SFT model（1000 条冷启动数据）做的 RL；三种设置对比：base model（探索空间大但格式差）> long CoT model（格式好但初始输出过长）> instruct model（思考固化、探索空间小）
- **误区2**：复现靠 response_len 稳定增长 ——response_len 和 explore_space 正比，希望的是模型探索出更有价值的思考逻辑，而非单纯拉长；reflection pattern 是否有效要看带反思 token 的 response accuracy 是否真的更高
- **误区3**：只关注 reward/loss/response_len ——应记录更多指标：输出同质化（entropy、编辑距离、N-gram）、各条件下 ACC、异常现象占比（无格式/超长/复读/中英混杂）、clip 发生频率
- **误区4**：为某个指标好看偏离实验目标 ——rlhf 目标是把 path@N 答对一次训成 path@3 答对一次，把收益集中在 path@1；response 多样性低大概率是 prompt 太难或太简单，换数据而非改 loss

### 核心结论

- **long CoT 实验的两个真正目标：reward 稳定提高 + response 探索出高级思考 pattern**；response_len 增长和 aha moment 不是关键
- o1/r1 真正牛的地方：把 code/math 上学到的 long CoT 能力泛化到通用能力

## 关键引文

> 目前的 long cot 实验重点是两个目标：reward 稳定提高，response 探索出一些高级的思考 pattern。至于什么 response_len 持续增长，什么 aha moment，并不是实验的关键！要知道，o1 和 r1 目前更擅长的其实是 planning，而不是无休止的"wait、but、however" `[重点/高亮]`
