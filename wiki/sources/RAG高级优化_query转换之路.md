## 摘要

准确找到与用户查询最相关的信息是 RAG 系统成功的关键。本文介绍三种 query 理解方法以增强检索过程：查询重写、Step-back 提示、子查询分解。每种技术通过修改或扩展原始查询提高检索信息的相关性和全面性。

## 核心内容

### Query 转化的优点

- **提升相关性**：查询重写有助于检索更具体相关的信息
- **更好的上下文**：后退提示允许检索更广泛的上下文和背景信息
- **综合结果**：子查询分解支持检索复杂查询的不同方面
- **灵活性**：每种技术可单独或结合使用，取决于用例

### 1. 查询重写（Query Rewrite）

- **目的**：使查询更具体详细，提高检索相关信息的可能性
- **方案**：重写不仅与原查询相似，还提供不同角度或视角，提高生成质量和深度
- 示例："气候变化对环境的影响是什么?" → "气候变化对各种生态系统的具体影响是什么，包括温度、降水模式、海平面和生物多样性的变化?"
- Prompt 要求 LLM 重新表述查询使其更具体、详细

### 2. Step-back 提示（退步提示）

- **目的**：生成更广泛、更通用的查询，帮助检索相关背景信息
- **方案**：通过考虑高层次概念和原则解决复杂问题，与直接解决形成对比
- "抽象的目的不是让你更迷糊，而是创建绝对精确的新语义层次"
- 示例："气候变化对环境的影响是什么?" → "气候变化的一般影响是什么?"

### 3. 子查询分解（Subquery Decomposition）

- **目的**：将复杂查询分解为更简单子查询，更全面检索信息
- **方案**：将复杂问题分解成一系列更简单子问题，依次解决，每个子问题受益于之前的答案
- 示例分解为：生物多样性/生态系统、海洋环境、天气模式、陆地环境等子问题
- Prompt 要求分解为 2-4 个子查询，综合回答时覆盖原始查询

### 实现要点

- 所有技术都使用大模型进行查询转换
- 自定义提示模板指导模型生成适当转换
- 每种转换提供单独功能，易集成到现有 RAG 系统
