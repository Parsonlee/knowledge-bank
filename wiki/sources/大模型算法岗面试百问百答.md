## 核心要点

### RAG 技术体系

- **RAG 总体流程**：数据预处理 → 分块（关键步骤）→ 文本向量化 → query 向量化 → 向量检索 → 重排 → query+检索内容输入 LLM → 输出
- **外挂知识库解决的问题**：克服遗忘问题；提升回答准确性/权威性/时效性；解决小众领域缺乏涉猎；提高可控性和可解释性
- **RAG 效果评估**：
  - 检索环节：MMR（平均倒排率）、Hits Rate（命中率）、NDCG
  - 生成环节：非量化（完整性/正确性/相关性）、量化（Rouge-L）
- **RAG 改进方向**：query 侧纠错/改写/规范化；对向量数据库做层次索引；对 LLM 垂直领域微调；对最终输出做后处理
- **常见向量检索算法**：ANN（乘积向量/暴力搜索/hnswlib）、KD 树

### LLM 架构与问题

- **幻觉问题**：生成内容无意义或不忠实于源内容
  - 原因：训练数据与源数据不一致/数据没对齐/编解码器缺陷；用户问题超出模型认知范围
  - 解决：引入外挂知识库、加入纠偏规则、限制输出长度
- **复读机问题**：重复生成某些话
  - 原因：数据质量不高/大量单一重复文本；文本过长时 greedy search 导致循环重复
  - 解决：丰富数据集多样性；同义词替换数据增强；温度参数调整；后处理与过滤
- **主流开源大模型 LLaMA 架构改动**（基于 Transformer）：
  - RMSNorm 前置归一化（替代 LayerNorm，受 GPT-3 启发）
  - SwiGLU 激活函数替换 ReLU（受 PaLM 启发）
  - 旋转位置嵌入 RoPE（替代绝对位置嵌入，受 GPT-Neo 启发）
  - 高效 causal multi-head attention 减少内存占用

### SFT 方法

- 全微调
- Adapter Tuning
- Prefix Tuning
- Prompt Tuning
- P-Tuning v1
- LoRA
- RLHF
- **SFT 学习率设置**：预训练阶段的 10%，通常效果较好

### LoRA 原理

- 在原始 PLM 旁边增加旁路，做降维（矩阵 A，随机高斯初始化）再升维（矩阵 B，0 矩阵初始化）
- 训练时固定 PLM 参数，只训练 A 和 B；输出时叠加 BA 到 PLM 参数
- 保证训练开始旁路矩阵为 0，不影响初始行为
