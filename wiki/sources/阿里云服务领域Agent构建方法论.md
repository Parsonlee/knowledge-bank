## 核心要点

### 1. Agent 概念之争

**三组核心争论**：
- 智能体 vs 代理：是否必须由 LLM 驱动，是否需要"智能"
- 自主规划 vs 工作流：是否必须自主规划，Workflow 是否算 Agent
- 函数调用 vs 角色扮演：是否必须函数调用，人设是否为 Agent 属性

**吴恩达观点**：agenticness 是一个光谱，不同程度的自主性都是合理的，无需纠结是否"真正是 Agent"。

**本文框架**：
- **狭义 Agent**：理想状态，能自主规划/分解/执行/反思/决策，现阶段是"技术理想主义"
- **广义 Agent**：以帮助人自动化完成任务为目标，可混合大模型/小模型/硬编码/规则引擎

### 2. Agent 落地四大挑战

| 挑战 | 描述 |
|------|------|
| 运行效果不稳定 | 提示词书写不当：过短（脑补）/过长（注意力失焦）/存在冲突矛盾 |
| 规划如何平衡 | LLM 自主规划（智能但不可控）vs Workflow（可控但成本高/固化） |
| 领域信息集成 | 通用模型不具备领域垂类知识，如何注入 |
| Agent 响应速度 | 大参数慢/小参数效果差，性能与效果反比 |

### 3. 提示词调优方法

**常见问题**：过短（表意模糊）/ 过长（注意力失焦）/ 存在冲突矛盾

**解决路径**：
1. 提供提示词模板，让团队遵循
2. **AI 辅助生成与调优**：将模板给 AI，结合需求先写一版，AI 检测冲突矛盾，持续迭代——"最懂大模型的是大模型自己"

### 4. 规划层：Workflow vs LLM 自主规划的权衡

**规律**：LLM 自主规划 ↑ 智能化 ↑ 但可控性 ↓；Workflow ↑ 可控性 ↑ 但智能化 ↓

**适合 Workflow 的场景**（标准化、重复执行）：
- 订单财务类：查询是否过期、计算退订费用（数学计算交给大模型易幻觉）
- 排班类：定时推送排班提醒

**适合 LLM 自主规划的场景**（探索化、问题复杂）：
- RDS 数据库异常诊断：用户只说"查询很慢"，需逐步自主排查会话连接/QPS/内存/慢SQL
- Agentic RAG：长尾产品无现成知识库，自主生成 query → 反思 → 重新搜索

**中间状态：Multi-Agent**（内部稳定可控 + 外部自主灵活）：
- 例：邮箱收发信诊断——内部排查流程用 Workflow，外层用 LLM 动态决策调用哪个 Workflow

### 5. Workflow 三代演化路径

| 代 | 模式 | 优点 | 缺点 |
|----|------|------|------|
| 第一代 | 自然语言编排 + LLM 逐步执行（不完备流程图） | 构建成本低，允许不完备 | 速度慢（每环节 LLM 决策），只能从头到尾 |
| 第二代 | 代码/大模型混编 + 规则驱动按步执行（完备流程图）| 速度快，稳定性高 | 必须完备流程图，门槛高，难以中途进入或灵活跳转 |
| 第三代 | 自然语言编排 + LLM 自主规划执行（不完备流程图作 RAG 参考）| 灵活智能，非从头到尾 | — |

**现状**：三种模式并存，具体场景具体选择，无完美方案。

### 6. 领域数据集成三方法

1. **Prompt 动态领域要求**：类 RAG 方式，按场景匹配度动态加载领域先验知识，不占太多上下文
2. **引入外部技能**：调用领域工具/知识库/文档，LLM 自主选择调用
3. **领域大模型训练**：构造领域数据（工具调用/多步/反问/Multi-Agent/拒识等语料）→ SFT/DPO/GRPO；评估：工具选择准确率/动作执行准确率/参数提取准确率

### 7. 响应速度优化三方法

1. **代码参数预转换**：非必要 LLM 模块转换为代码/脚本
2. **推理加速**：模型量化（Int4/Int8）/ 优化 KV Cache / FlashAttention / vLLM / 高性能 GPU
3. **降低参数量**：用大参数模型蒸馏小参数模型，高频场景（Function Call）用小模型

### 8. Agent 构建与持续调优路径图

```
提示词原型 → 提示词调优
    ↓（仍不稳定）
拆解 Workflow → 代码规则编排
    ↓（固化不灵活 or 搞不定）
Multi-Agent 架构 → 端到端测试
    ↓（仍无法满足准确率/速度/稳定性极高要求）
模型定制化训练与调优
```

成本随路径递增，效果也随之提升；技术选型取决于场景要求。

## 关键引文

> "最懂大模型的是大模型自己，所以我们可以先搞清楚大模型的'脑回路'，才能更好的让大模型理解我们的需求。" [重点/高亮]

> "LLM 自主规划的智能化程度越高，可控性越低，而 Workflow 编排的可控性越高，则智能化则越低。" [重点/高亮]

> "狭义的 Agent 虽然构建起来很简单，任务要求全盘交给大模型来做即可，但是效果的稳定性不佳，落地过程中有太多的问题。" [高亮]
