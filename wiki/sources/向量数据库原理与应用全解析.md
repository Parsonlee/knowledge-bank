# 向量数据库原理与应用全解析

## 摘要

Eternity（Datawhale 成员）撰写的向量数据库综述，从多模态向量化的技术演进讲起，系统介绍 [[概念_向量数据库|向量数据库]] 的概念、动机、五类索引方法、相似度度量、前后处理以及主流产品选型与系统优化趋势。

## 核心内容

### 向量数据库概念

- 通过 [[概念_Embedding与向量检索|Embedding 模型]] 将图像/文本/音频转为向量化表示进行存储，支持快速相似度检索。

### 文本向量化演进

- BoW / TF-IDF → Word2Vec（2013，CBOW / Skip-gram，静态）→ GloVe（2014，全局共现矩阵）→ [[概念_Transformer架构|Transformer]]（2017）→ BERT（2018，双向、masked）→ GPT 系列（自回归）。

### 图像向量化演进

- SIFT / HOG → AlexNet（2012）/ VGG（2014）/ [[实体_ResNet|ResNet]]（2015）→ [[实体_ViT|ViT]]（2020）→ [[实体_CLIP|CLIP]]（2021，对比学习对齐视觉+语言）。

### 音频向量化演进

- MFCC → Wav2Vec（2019）→ Wav2Vec 2.0（2020，Transformer + 向量量化 + masked prediction）。

### 为什么需要向量数据库

- 传统数据库（B-tree / Hash）无法做"语义相似"查询。
- LLM 之前的应用：人脸识别、推荐系统（Spotify Annoy → Voyager HNSW）、商品搜索、异常检测。
- LLM 时代：RAG 引入私有/实时知识，缓解上下文窗口限制。

### 索引方法

- **Flat（暴力检索）**：100% 召回，慢。
- **Tree-based（树）**：KD-Tree，受维度灾难影响。
- **Hash-based（哈希）**：LSH，亚线性查询。
- **Quantization-based（量化）**：IVF-PQ，聚类 + 乘积量化。
- **Graph-based（图）**：[[实体_HNSW|HNSW]]，多层可导航图，毫秒级，工业界标准。

### 相似度度量

- 欧氏距离（L2）、曼哈顿距离（L1）、切比雪夫距离（L∞）、余弦相似度、内积（点积）、汉明距离（二值）、Jaccard 相似度（集合）。

### 前处理与后处理

- **前处理**：Embedding 生成、归一化、去重、PCA 降维。
- **后处理**：对候选集做精确距离重排序、元数据过滤、多样性重排序。

### 主流产品

- **Faiss**（Facebook）：库而非完整数据库，支持 GPU，无分布式。
- **Chroma**：轻量、Python、LangChain 默认，不支持分布式。
- **Milvus**、**Weaviate**、**Pinecone**、**Qdrant**。

### 系统优化趋势

- 新 ANN 算法（ScaNN）、量化技术进步、硬件加速（GPU / FPGA / ASIC）、分布式架构、实时更新、内存成本控制。
