## 核心要点

### 记忆分类体系（翁荔定义框架）

翁荔（Lilian Weng）定义：记忆 = 获取 + 存储 + 保留 + 随后检索信息的过程。

三层类型：
1. **感官记忆**：原始输入的嵌入表示（文本/图像/多模态）
2. **短期记忆（上下文内记忆）**：Transformer 上下文窗口内，有限且临时
3. **长期记忆**：跨对话可检索的外部向量存储

### 五类 LLM 记忆形式

| 类型 | 机制 | 优点 | 缺点 |
|------|------|------|------|
| 上下文内记忆（短期）| 信息放入 prompt | 通用直接 | 有限、临时、高成本 |
| 外部记忆（非参数化）| 向量数据库 + RAG | 灵活、无需重训 | 检索准确性关键 |
| 参数化记忆 | 微调/知识编辑/LoRA | 推理快、无额外检索 | 更新成本高、灾难性遗忘 |
| 分层式记忆 | 情景记忆 + 语义记忆分层 | 支持反思/抽象 | 架构复杂 |
| 混合记忆 | 以上组合使用 | 兼顾稳定性与灵活性 | — |

### 主流大厂实现

- **ChatGPT**：结构化片段存于服务器（向量DB + 常规DB），提示工程注入，用户可管理；跨会话历史引用，越用越懂你
- **Claude**：仅在用户要求时检索过去聊天，不建用户档案，更简单
- **Gemini**：类似 ChatGPT，另支持用户自行录入记忆

### 前沿记忆系统

**MemGPT**（arXiv:2310.08560）：
- 专用记忆 LLM 管理工作 LLM 上下文窗口
- 主上下文 = 系统指令 + 工作上下文 + FIFO 队列
- 函数在主上下文与外部上下文（归档/召回数据库）间移动数据
- `request_heartbeat=True` 链接多步检索

**MemOS**（记忆张量科技）：
- 三层：API接口层 / 记忆调度管理层 / 存储基础设施层
- 标准化 **MemCube** 单元：明文 + 激活状态 + 参数记忆统一调度
- 支持下一场景预测预加载记忆，降低响应延迟
- 可跨模型记忆迁移复用

**MemoryOS**（北邮百家AI）：
- 段页式三级存储 + 四大核心模块（存储/更新/检索/生成）

**MIRIX**：
- 6类记忆：核心记忆、情景记忆、语义记忆、程序记忆、资源记忆、知识金库
- 元记忆管理器（Meta Memory Manager）+ 多个专用 Memory Manager
- 检索流：粗检索→目标检索选择→精细检索→结果整合

**G-Memory**（NUS+同济大学）：
- 三层图式：洞察图（单智能体主观理解）/ 查询图（任务意图）/ 交互图（协作路径/记忆共享权限）
- 解决多智能体记忆污染和路径冲突

**M3-Agent**（字节+浙大+上交）：
- 多模态长期记忆智能体
- 记忆阶段：实时处理视频/音频 → 情节记忆 + 语义记忆
- 控制阶段：基于长期记忆推理执行
- 强化学习驱动多轮推理迭代检索

### 参数化记忆前沿

**Meta Memory Layers**（arXiv:2412.09764）：
- 用键值对检索替换 FFN，实现模型内部"查表式记忆"
- 记忆容量达 1280 亿参数，提升两个数量级

**Branch-Train-MiX（BTX）**：
- Branch：从种子模型创建多专家副本，各自独立训练不同领域
- MiX：专家前向子层合并为 MoE 模块
- 微调阶段训练路由网络

**Yan 2.0 Preview**（RockAI）：
- 非 Transformer 架构的原生记忆能力
- 可微分神经网络记忆单元实现存储/检索/遗忘

### 五大挑战与趋势

1. **遗忘与整合**：当前系统"只进不出"，缺乏智能遗忘机制；信息冲突更新难
2. **效率-成本-可扩展不可能三角**：外部记忆有检索延迟，参数记忆更新昂贵
3. **多模态记忆**：从纯文本扩展到视频/音频（LVMM、M3-Agent）
4. **多智能体共享记忆**：智能体集群共享记忆池，集体智能 vs 隐私保护
5. **终极目标：自动演化**：无需人工干预的终身学习（《Long Term Memory : The Foundation of AI Self-Evolution》）

### 结论

记忆正将 LLM 从无状态"文本计算器"转变为有状态"认知主体"。未来综合记忆架构三大特征：多模态原生 / 终身自主演化 / 社会化与协同。
