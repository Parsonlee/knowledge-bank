# 美团搜索查询改写实践

## 摘要

美团搜索场景下查询改写（Query Rewriting / Query Expansion）技术迭代全景：从离线候选挖掘、语义判别模型、到在线生成改写。查询改写控制召回语法中的文本，在全部流量上生效，直接影响无结果率、召回结果数、搜索点击率。最终美团App搜索有改写流量占比约73%。

## [重点/高亮] 查询改写定义与应用方式

用户高亮内容：

> 用户端拓展用户的查询词——即查询改写，或Document端拓展文档关键词——即Document标签

> 查询改写（Query Rewriting，或称为查询扩展Query Expansion）。查询改写的应用方式是对原始Query拓展出与用户需求关联度高的改写词，多个改写词与用户搜索词一起做检索，从而用更好的表述，帮用户搜到更多符合需求的商户、商品和服务。

## 美团场景特殊挑战

- 本地生活服务：用户需求和商家面向本地，供给相对较少
- 多业务聚合：团购/外卖/买菜/优选等多聚块，不能依赖排序解决相关性
- 语言歧义多：短Query歧义、相同Query不同城市含义不同
- 认知关联性：用户"找店"认知，需场景关联知识（如"配眼镜"="眼镜店"）
- 商户不做SEO：文本不匹配漏召回问题更严重

## 离线候选挖掘

### 搜索日志挖掘
- 点击共现：两个点击相同Document的Query构建相关关系（噪声大）
- Session挖掘：用户连续输入的Query构建相关关系（泛化强但切割难）
- 词对齐：借鉴翻译思想，Query召回商户标题去除商户名后做平行语料
- 商户/商品内SEO：商品上架时的关键词堆砌（噪声大）

### 基于图方法挖掘
- SimRank++：同构信息网络相似度量算法，优化构图（Wilson平滑边权重 + Session换词Query作为Document节点）
- 优化后数据量提升30%，准确率72%→83%
- DeepWalk随机游走：建立间接关系的Query联系，解决共现无法传递的问题

### 基于语义向量挖掘
- Word2Vec/Doc2Vec/Session序列：不同上下文结构训练Embedding
- FastText训练词向量（解决OOV）+ LSH/DSSM向量检索 + XGBoost过滤
- Sentence-BERT/SimCSE：有监督训练，Fine-Tuning双塔结构，Faiss亿级检索
- 准确率高，冷启动友好，无需统计特征和平行语料

## BERT语义判别模型

### 通用判别
- NMT-BERT协同训练（半监督）：BERT产出NMT训练数据→NMT生成改写→BERT再训练，迭代3轮收敛
- 关系抽取挖掘困难负例：UGC中"如A、B、C等"句式提取同位词负例
- 四阶段训练：无监督Continue Train→半监督Co-training→样本增强→人工标注Tuning
- 最终准确率94%+

### 商品领域判别
- 训练数据：SPU共现/向量召回/质检标注 + 类目/点击/UGC困难负例
- 数据增强：Random/Batch/Hard Sample Negatives
- 模型优化：R-Drop + Child-tuning，F1提升2.5PP
- 图向量融合：GCN/GAT Graph Embedding融入BERT，F1提升1.6PP

## 在线服务

### SMT统计翻译模型
- 噪声信道模型：对齐字典(BERT过滤) + 结构化Decode(BeamSearch) + XGBoost排序（语义+有效性）
- 上线后有改写流量覆盖提升12%

### NMT神经网络翻译模型（强化学习优化）
- 问题：覆盖长尾Query，分词粒度限制和自然语言类Query
- 预训练Seq2Seq + BERT初始化Encoder + Coverage机制（解决过翻）
- 强化学习框架：
  - Agent=NMT，Environment=离线搜索日志+BERT语义判别
  - Reward：召回商户重合度 + 用户点击商户重合度 + BERT语义分数
  - 加权归一化Reward叠加到Loss
- 效果：离线准确率69%→87%，解决语义改写/简写/输入错误/自然语言去词

### 在线向量化召回（模糊改写）
- 目标：商户名精搜文本不匹配的漏召回（换字/多字/少字）
- 双塔BERT：倒数第二层Avg Pooling，Binary Cross-Entropy Loss
- 负采样：Random + Batch(SimCSE) + Hard Sample Negatives
- 线上：前置流量划分→TFServing向量化→Antler ANN检索(基于Faiss, IVFFlat/HNSW)→后置规则过滤
- Faster-Transformer + FP16加速，多层缓存

## 平台化

- 数据精细化：区分同义/上下位/同位/不相关，不同用途（召回/推荐/排序负例）
- 算法工具化：挖掘工具 + 判别模型
- 在线服务可运维：快速接入/AB实验/干预
