## 核心要点

### LLM 时代核心业务资产重定义

LLM 时代的核心业务资产应该是：**Agent 的抽象、Tool 的抽象、业务 Prompt 设计、业务数据供给、微调后的模型、完整评测集**。

与之对应，Agent 平台需提供：Agent 注册与构建、Tool 接入、Prompt 调试、模型微调、测评等能力。

### Agent 定义

作者提出"广义 Agent"定义：**一个能够完成一定范围内人类任务的系统**，万物皆 Agent，即使没有 AI 参与的领域系统也可作为 Agent 参与互联。

Agent 规划角色演进：从人类编排 → 人类描述详细任务 → 人类发出简单指令，三个阶段会长久并存。

### 多 Agent 协作机制

参照微服务演进路径（单体→微服务），Agent 系统也会经历单 Agent → 多 Agent 协作的演进：

| 维度 | 机制 |
|------|------|
| 任务分配 | 集中式分配、分布式协商、基于角色划分、动态重分配 |
| 协作方式 | 平行协作、层级协作（管理 Agent 统筹）、专家协作 |
| 冲突解决 | 优先级决策、投票机制、仲裁 Agent、基于规则 |

核心模型：业务主 Agent → 任务中心恢复上下文 → Agent 仓库找协同 Agent → 多 Agent 交互协议 → 结果决策。

**MCP 工具服务也可抽象成 Agent**，并在这一层做鉴权/参数校验等附加逻辑。

### MCP 协议工程补充

当前 MCP 协议偏薄，集团视角需补充：

1. **公网/内网权限隔离**：内网 MCP 服务需统一网关收口，按请求类型给出相应工具集
2. **用户态权限控制**：MCP 协议缺少用户态信息，需补充用户级鉴权；用户无权限的工具应对大模型不可见
3. **工具快速接入**：HSF 支持 0 代码一键转 MCP Server；SLS/ODPS/IDEAs 等场景快速接入
4. **长工具列表优化**：二阶段工具匹配——基于向量相似性或 LLM 前置过滤无关工具，减少上下文长度并提升匹配准确性

### Agent 框架配套

除协作构建和 MCP 工具调用，Agent 平台还需：问题理解、关键记忆召回、知识库增强、评测、模型市场、记忆管理、链路追踪、评测管理、微调能力。
