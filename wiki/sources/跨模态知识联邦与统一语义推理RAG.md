# 跨模态知识联邦与统一语义推理RAG

## 来源信息

- **标题**：跨模态知识联邦与统一语义推理RAG
- **作者**：王传阳 / 枫清科技
- **发布时间**：2025-06-10

> QCon 演讲实录，解决企业级复杂场景下异构知识的结构离散性和模态多样性问题。

## [重点/高亮]

> 融合知识库它主要是将左边散落的各种多元异构数据，包括指标、结构化数据、文档、图片等信息，通过知识融合技术整合到一个统一的融合知识库中。知识融合的过程涉及多种常用技术手段。

> 从技术形态和产品形态来看，融合知识库是一个逻辑概念。底层的物理存储保持不变，例如文档仍然存储在向量数据库中，关系型数据库的数据源（如 MySQL 或 PostgreSQL）仍然保留在原处，只是它们的元数据被抽取出来，以逻辑视图的形式存储在融合知识库中。

## 主要内容

### 复杂场景挑战

1. **结构的离散性**：相同主题知识分散在不同文档/媒介中；同一文档内部不同位置引用同一实体（如"张三"与"小三子"）
2. **模态的多样性**：文本、音视频、图片、关系型数据库、表格等多种形式
   - 多模态检索：文字检索出图片/视频/表格
   - 跨模态检索：一种模态检索另一种模态

### 核心组件一：融合知识库

- 逻辑概念，底层物理存储不变
- 将多元异构数据（指标、结构化数据、文档、图片）通过知识融合整合
- 产品形态：用户可创建文档知识库、网页知识库、数据源知识库、指标知识库，或创建融合知识库统一组织
- 可指定关注的领域模型（实体和关系），为统一语义层铺垫

### 核心组件二：统一知识图谱

#### 图谱生成

- 文档数据：Graph RAG Light 提取实体和关系（指定关注的实体/关系降低成本）
- 图片/视频：多模态大模型或已有标注系统提取元信息
- 关系型数据库：提取元数据（表用途、字段业务含义）
- 涉及实体合并和关系发现

#### 图谱检索

- 用户问题 → 向量库中实体/关系匹配（语义搜索）
- N 度扩展查询（二跳/三跳）提取相关子图
- 子图包含多模态信息 → 从底层物理存储提取具体数据 → 送入大模型生成答案

### 与 GraphRAG 对比

- 优化：预先指定领域模型/实体/关系，让抽取更聚焦、减少算力消耗
- 横向扩展：加入元数据提取、多模态图片视频识别，构建含多模态信息的知识图谱

### 案例一：医院电子病历查询

- 融合知识库：结构化数据（患者信息/医嘱/住院记录）+ 电子病历文本
- 统一知识图谱：以患者为中心，关联病症/治疗措施/效果
- 应用：医生输入症状 → 系统检索历史相似患者治疗方案

### 案例二：银行风险监管指标分析

- 融合知识库：指标库 + 文档库
- 应用：查询"不良贷款率"时，不仅返回数据，还检索相关规定文档给出预警

### 未来方向

- 统一语义层的动态更新（实时图更新难题）
- 图像/视频数据高效处理（目前仅元数据阶段）
- 行业语义模型与行业图谱共享
- 融合知识库的标准化和场景化
