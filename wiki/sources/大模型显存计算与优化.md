# 大模型显存计算与优化

## 来源信息

- 标题：[LLM]大模型显存计算公式与优化

- 作者：kaiyuan（知势榜科技互联网领域）

## 核心要点

1. **[重点/高亮] 显存消耗组成**：训练/推理时 GPU 显存消耗内容包括——模型参数（parameter）、优化器状态值（optimizer_state）、激活值（activation）、梯度值（gradient）、输出数据（input）、临时变量（temporary）、自动梯度（autograd_detail）、未知数量（unknown）。其中用户可估算的为前五项，未命名数据（temporary/unknown）可导致估算误差超过 30%。

2. **模型显存计算**：`ModelMem = TypeSize * Params`。fp32 下 1B 参数 ≈ 4GB；fp16/bf16 下 1B ≈ 2GB；fp8/int8 下 1B ≈ 1GB。近似规则：1B 参数 fp32 ≈ 4GB。

3. **Adam 优化器状态**：混合精度训练中 Adam 需要模型副本(4B) + Momentum(4B) + Variance(4B) = 12 Bytes/参数。8 位优化器为 4+1+1 = 6 Bytes/参数。

4. **激活值公式（Megatron 论文）**：`Activation = s*b*h*(34 + 5*a*s/h) * L * γ`，其中 s=序列长度, b=微批量, h=隐藏维度, a=注意力头数, L=层数, γ 为精度系数。

5. **并行策略降低显存**：TP（Tensor并行）、SP（序列并行）、PP（Pipeline并行）、Zero（1/2/3）、重计算。3D 并行下模型显存 ≈ Model/(PP*TP)，梯度显存 ≈ Gradient/PP（梯度不除 TP 因反向需 AllGather 完整梯度）。

6. **推理显存估算**：`InferMemory ≈ 1.2 * ModelMemory`，比训练场景简单。

7. **显存优化路径**：多卡并行 → 算子/数据类型优化 → 消除框架副本 → 显存管理 → 底层 API。核心思路为"时间换空间"（如重计算，消耗算力）和"空间转移"（如多卡并行/offload，消耗 I/O 带宽）。

## 关键引文

> 显存消耗的内容包括：模型参数（parameter）、优化器状态值（optimizer_state）、激活值（activation）、梯度值（gradient）、输出数据（input）、临时变量（temporary）、自动梯度（autograd_detail）、未知数量（unknown)。
