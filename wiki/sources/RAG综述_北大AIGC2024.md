# RAG综述_北大AIGC2024

## 摘要

北京大学崔斌教授团队 PKU-DAIR 系统调研近 300 篇 RAG 相关论文，发布综述 "Retrieval-Augmented Generation for AI-Generated Content: A Survey"（arXiv:2402.19473）。覆盖 RAG 基础方法、提升方法、多模态应用（文本/代码/音频/图像/视频/3D/结构化知识/AI4S）、评估基准及局限性与未来方向。

## RAG 四大基础范式

根据检索器如何与生成器交互分为四类：

### 1. 基于查询的 RAG（Query-based）
- 也称提示扩充（Prompt Augmentation）
- 检索到的内容与原始查询合并为复合输入送入生成模型
- 最广泛采用的方法
- 可直接与提示词工程、少样本学习相结合

### 2. 基于隐空间表达的 RAG（Latent Representation-based）
- 检索内容在生成模型内部以隐空间表达方式进行交互
- 边生成边交互，提升理解能力与生成质量
- 交互方法：简单拼接、注意力机制等

### 3. 基于概率表示的 RAG（Logit-based）
- 解码阶段利用概率方法融合检索信息
- 对检索结果的概率分布进行求和或融合
- 模型在生成时权衡不同检索信息的相关性和可信度

### 4. 投机 RAG（Speculative RAG）
- 利用检索过程替代部分或全部生成过程
- 当检索成本低于生成成本时具有应用潜力
- 应用：投机推理中检索替代小模型生成、搜索过往相同问题直接回答

## RAG 五类提升方法

### 1. 输入提升
- **查询改写（Query Transformation）**：修改输入查询提高检索结果
- **数据增强（Data Augmentation）**：去除无关信息、消除歧义、更新过时文档、合成新数据

### 2. 检索器提升
- **递归检索（Recursive Retrieve）**：拆分查询多次搜索检索更高质量内容
- **块优化（Chunk Optimization）**：调整块大小获得更好检索结果
- **检索器微调（Finetune Retriever）**：嵌入模型微调提升领域性能
- **混合检索（Hybrid Retrieve）**：统计词频+向量相似性多种检索器结合
- **重排序（Re-ranking）**：检索后重新排序实现多样性和更好结果
- **元数据过滤（Meta-data Filtering）**：用元数据（时间/目的等）过滤筛选

### 3. 生成器提升
- **提示词工程（Prompt Engineering）**：压缩/回退/主动/思维链提示
- **解码过程控制（Decoding Tuning）**：超参调整、限制输出词汇表
- **生成器微调（Finetune Generator）**：更精确领域知识、更好匹配检索器

### 4. 结果提升
- **结果改写（Rewrite Output）**：另一个生成模型作为编辑器/改写器

### 5. RAG 流程提升
- **自适应检索（Adaptive Retrieval）**：基于规则（token 概率/困惑度）或基于模型判断是否需要检索
- **迭代 RAG（Iterative RAG）**：当前输出反映缺乏知识，检索缺失信息作为下一轮上下文，循环直到达标

## 评估基准

### 主要评判标准
- **忠实性**：从检索内容中推断正确答案的能力
- **答案相关性**：生成结果是否解决了问题
- **上下文相关性**：检索内容是否包含足够知识回答查询

### 其他评判指标
- **噪声稳健性**：从噪声文档中提取信息的能力
- **拒绝回答能力**：内容不足时是否拒绝响应
- **信息整合能力**：整合多个检索内容获取知识的能力
- **反事实稳健性**：识别检索内容中反事实错误的能力

## 局限性

1. **检索结果中的噪声**：ANN 只能提供近似匹配，噪声有时反而增强生成质量（多样性），影响机制仍是开放问题
2. **额外开销**：引入检索和交互步骤导致处理延迟，递归检索/迭代 RAG 开销更突出
3. **检索与生成复杂交互**：不同目标、不同潜在空间，协调是挑战；分离方法更模块化，集成方法可从联合训练获益
4. **长上下文生成挑战**：长上下文模型不能取代 RAG（动态信息/长尾知识灵活性），未来 RAG 将利用长上下文能力提升性能

## 未来方向

1. 探索更先进的 RAG 基础方法和提升方法
2. 更高效部署 RAG 系统（当前仅 LangChain/LlamaIndex 方案成熟）
3. 更好发挥结合长尾和实时知识的能力，设计不断更新知识源的 RAG 系统
4. 将 RAG 与微调、强化学习、思维链、Agent 等其他 AIGC 技术结合
