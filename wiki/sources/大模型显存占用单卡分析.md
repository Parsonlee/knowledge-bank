## 核心要点

### 数据精度基础
- 1 参数 FP32 = 4 bytes；BF16/FP16 = 2 bytes；NF4 = 0.5 bytes
- BF16：指数位宽与 FP32 相同（范围大），小数位少（精度低），业界训练首选
- TF32 实际只有 19 bit（不是 32）

### 混合精度训练（FP32+BF16，AdamW）
- 流程：备份 FP32 权重 → 转 BF16 → forward/backward（FP16梯度）→ FP32 更新权重 → 循环
- 显存组成：FP32 模型副本（4Φ）+ BF16 模型（2Φ）+ BF16 梯度（2Φ）+ FP32 Adam 一阶+二阶矩（8Φ）= **约 16 bytes/参数**
- 激活值和梯度是动态显存，面试时可忽略激活值
- 实测：llama3.1 8B ≈ (48+16+64) = 128G（不含激活值）

### 推理与 KV Cache
- 推理显存主要是模型参数本身
- KV Cache 公式：`Memory = batchsize × seqlength × hiddensize × layers × 2(KV) × 2(bytes)`
- llama7B，batch=64，seq=2048：KV Cache ≈ 68G；batch=1 时约 1G
- **为何没有 Q Cache**：Self-Attention 中每个位置 t 的输出只依赖当前 Q_t，无需缓存历史 Q
- MQA/GQA：共享 KV 头，GQA 公式中 `hiddensize → group × head_dim`，业界主流方案

### LoRA 显存
- 只训练旁路低秩矩阵，原模型不更新梯度，优化器只管 LoRA 参数（可忽略）
- 总显存 ≈ **2Φ**（只需加载 BF16 原始模型），7B 模型约 14G

### QLoRA 显存
- 存储参数 NF4（4bit），计算时反量化为 BF16，用完释放
- 训练比 LoRA 多约 30% 时间；总显存 ≈ **0.5Φ**
- 三个创新：NF4量化（正态分布假设）、双重量化、优化器分页（防OOM）

### 总结表

| 训练方式 | 显存约估 |
|---------|---------|
| 全参微调（全FP16） | 8 bytes/参数 |
| 全参微调（BF16混合精度） | 16 bytes/参数 |
| LoRA | 2 bytes/参数 |
| QLoRA | 0.5 bytes/参数 |
