## 核心要点

### 问题：AI 编程的"80分困境"

AI 能完成 80% 的基础工作，但剩余 20% 的精细调整（业务规则/异常处理规范/日志格式/VIP 特殊逻辑等）反而比从零开始更耗时。根本原因是**信息不对称**——AI 就像没有经过入职培训的新员工。

### 第一板斧：给 AI 做"入职培训"

用"代码解构与业务分析师 Prompt"深度分析现有系统，生成私域知识库：

```
私域知识
├── 01_架构概览.md
├── 02_数据模型.md
├── 03_业务规则.md
├── 04_术语词汇.md
├── 05_开发规范.md
└── 06_常见问题.md
```

代码解构 Prompt 核心能力：
- 系统分析师：追踪跨模块调用链（深度≥3层），展开抽象类所有实现子类
- 业务洞察顾问：从技术实现反向推导业务规则，发现隐式决策点
- 可视化输出：架构图、类图、时序图、数据库关系图、业务逻辑公式手册

### 第二板斧：基于知识库的智能编程

"开发专家 Prompt" + 私域知识库 → 一次性生成符合项目规范代码

传统模式：5+ 轮调教才能达到要求
知识工程模式：AI 基于私域知识直接输出完整 Controller/Service/DTO 层 + 异常处理 + 业务校验 + 日志埋点 + 单测

开发专家 Prompt 核心：20年 Java 经验人设，ultrathink + 批判性思考/反面考虑/复盘各3轮，优先检索本地 markdown 知识文件。

### 第三板斧：私域知识的自动维护

"文档自动维护 Prompt"：
1. 执行 `git diff` 获取变更（Java/Python/JS/配置文件）
2. 读取文档维护规范
3. 智能增量更新数据模型手册、术语词汇表
4. 质量校验（一致性/格式/完整性）
5. 自动更新文档并返回概要

形成**自我进化知识生态**：代码变更 → 自动更新文档 → AI 获得最新知识 → 生成更准确代码

### 私域知识工程意外收获

除提升 AI 编程效率外，还带来：系统文档自动化、新人入职加速、知识沉淀标准化。
