## 核心要点

### 阿里内部 AI Coding 探索（2023 年启动）

**当前成果**（内部指标）：
- 日活约 12,000 人，集团内部渗透率约 65%
- 代码补全整体采纳率约 28%（IntelliJ IDEA 约 33%）
- 约 20% 的 Code Review 问题由 AI 生成

**已落地能力**：代码补全、代码会话、Prompt 市场、Extensions（工具/流程编排）、Code Review（精确到行 + 一键修改建议）、代码总结。

### Copilot 阶段的瓶颈

四大核心痛点：
1. **工具理解能力有限**：难以理解用户真实意图的边界条件和约束
2. **用户需求描述不清晰**：用户表达随意，结果偏差即放弃
3. **领域知识整合困难**：同一概念命名不一致（item/product），难以转化为可学习内容
4. **用户使用成本高**：代码生成只达 85-90% 完成度，本地修改测试流程长

**模型层面缺陷**：大上下文时关键信息抓取能力变弱；缺乏逐步解决问题能力；推理模型速度慢 token 多。

**工程层面问题**：平台间数据不连通（需求/代码/发布/测试）；知识/文档质量参差，大量图片信息未解析；历史提交信息未规范化。

### 技术进步带来的转机

1. 产品进步（Cursor 多行补全、Apply 一键插入）
2. 多模态模型能力提升（理解架构图、截图理解用户意图）
3. 端到端代码生成能力增强（样式/布局/文字近原图）
4. 推理模型擅长复杂问题（了解自身能力边界，避免中途给错误结论）

### 从 Copilot 到 Agent 的转变

**规则驱动模式的局限**（基于规则的流程编排）：
- 泛化能力不足：需求/上下文变化即失效
- 成本高昂：每个场景单独构建流程
- 复杂任务难以应对：规则无法穷尽所有情况

**模型驱动 Agent 模式（参考 Devin）**：让 Agent 具备自主思考、自主规划、工具使用、反思和自我学习能力。

### 双 Agent 产品

**产品一：IDE Agent**
- 在 IDE 中提出需求，模型驱动调用 Shell/代码库搜索/文件编辑/读取等能力
- 典型场景：小需求实现、仓库理解、**场景左移**（Code Review/缺陷扫描在 IDE 中借助 Definition/Reference 工具发现深层问题）
- 逐步执行、每步反馈

**产品二：Aone Agent**（通用 Agent，类 Devin）
- 用户发出需求 → 后台启动容器 → 运行引擎 → 生成代码 → 容器内测试 → 反思 → 循环
- 目标：全研发全生命周期提效（技术调研、需求代码、前端代码+调试+预览）
- 架构：Leader Agent 分发任务 → 子 Agent 执行（terminal/shell/文件/Web/MCP）

### 通用 Agent 遇到的挑战

**工程挑战**：
- **记忆管理**：短期记忆防止 token 膨胀；长期记忆存储成功经验（如何选择性召回是难题）
- **评估困难**：长链路任务（百步以上）如何找到标准输入输出；结果评估 + 过程评估
- **工具使用**：浏览器控制台调试对 Agent 困难；工具间通信（主 Agent 未传凭据给子 Agent）

**模型层面挑战**：
- 指令跟随能力：明确禁止某操作仍坚持；不按规则测试而是盲目尝试
- 长上下文时隐藏细节信息捕捉能力弱
- 反思能力不足：误入歧途不知回头；陷入死循环

**战略风险**：
- 成本：使用 Claude 处理一个复杂任务约 50-80 元
- 隐私：外部闭源模型数据隐私问题
- 限流/被降智：外部模型提供商同时在做自己的 Agent 产品
- 安全/授权边界：异步执行不可控；授权边界模糊；责任归属问题
