## 来源信息

- 标题：自适应快慢思考推理模型（Adaptive Reasoning Model）：Qwen3混合思考->字节AdaCoT->清华AdaThinking
- URL：https://zhuanlan.zhihu.com/p/1910075005586346955

## 核心要点

### 背景
- Large Reasoning Model（LRM）具备 LongCoT 能力（Inference Time Scaling），但长思考 = 高延迟高成本
- 并非所有问题都需要长思考：简单问题快速回答，复杂问题才需"打草稿"
- **自适应快慢思考（fast-slow thinking）/ 混合思考（thinking-nonthinking mixed）** 是业界新方向，目标：减少不必要推理消耗而不损害效果

### 阿里 Qwen3 混合思考（SFT 方案）
- 思路来自 SFT 数据构造，在 user query 后加 `/think` 或 `/no_think` 标签
  - 有 `/think`：模型填充 `<think>详细思考</think> 回答`
  - 有 `/no_think`：模型输出 `<think></think> 回答`（空思考标签）
- thinking 数据由 stage2 reasoning model 对 stage1 数据做拒绝采样生成
- 涌现能力：**Thinking Budget** —— 接近 max_tokens 时拼接终止语让模型直接作答
- RL 阶段加 format-following 奖励强化遵循 `/think` / `/no_think` 指令
- 局限：模型不会**主动**判断是否需要思考，需人为在 prompt 中控制

### 字节 Seed AdaCoT（RL Pareto 方案）
- 把是否输出 CoT 转化为**多目标 Pareto 优化**：最小化 CoT 触发次数 T(θ) + 最大化模型得分 P(θ)
- SFT 冷启动：用额外模型判断每条数据是否需要深度思考，构造有/无 CoT 的混合训练数据
- RL 用 PPO 算法，奖励函数：`R = R_base - α1·P_miss - α2·P_over - γ·P_fmt`（惩罚漏思考和多余思考）
- **Selective Loss Masking**：mask 掉 `<think>` 后第一个 token 的 loss，避免模型坍缩为总是/从不输出 CoT `[重点/高亮]`
- 实验：65% CoT 触发率达到接近 100% 触发的效果；火山实际调用中仅 3.18% 需要 CoT

### 清华 AdaThinking（RL 约束优化方案）
- 前置分析：简单问题 CoT 无益甚至有害；越难的问题越需要更长思考；只有 Level 5 难度时 Thinking 才明显优于 Non-Thinking
- 优化目标：**约束优化** —— 最大化不思考率（y₁ = `</think>`），同时约束新模型 reward ≥ 参考模型 reward
- 通过拉格朗日乘数法转换为 PPO 优化
- **修改重要性采样**：因 thinking model 初始 y₁=`</think>` 概率为 0，直接修改 π_IS 使模型能探索"不思考"路径
- 更新条件：仅当不思考分数 + δ > 参考模型分数 且 > 思考模式分数时才更新 θ
- 实验：accuracy 提升，回复长度降低，消融实验验证修改重要性采样的必要性

### 三方案对比

| 方案 | 核心机制 | 谁控制思考决策 |
|------|---------|--------------|
| Qwen3 | SFT 混合数据 + RL format 奖励 | 人（prompt 标签） |
| AdaCoT | PPO Pareto 优化 + Selective Loss Masking | 模型自主 |
| AdaThinking | PPO 约束优化 + 修改重要性采样 | 模型自主 |

## 关键引文

> 模型在训练的过程中，如果处理不当，很可能会陷入到二元境地，比如一直都输出 CoT token 或者一直不输出 CoT token。这里的做法也很简单，在 RL 训练的过程中，把 policy gradient 中的 loss 不算所有的 token，而是 mask 掉 `<think>` 后的第一个 token。 `[重点/高亮]`
