# 阿里RAG技术演进

## 摘要

阿里云服务领域 RAG 技术从 0 到 1 建设总结，从发展趋势、技术挑战、核心举措、未来展望四个维度展开。核心目标：对客 Chatbot 答案准确率提升至 90%，小二 Copilot 信任度提升至 85%。提出 RAG 技术演进的四大核心命题：数据价值维度突破、异构检索跃迁、生成控制优化、评估体系重构。

## RAG 发展趋势

- 2024 年 RAG 采用率从 31% 上升至 51%，占据企业 AI 设计模式主导地位（Menlo Partners 调研 600 名美国企业 IT 决策者）
- 微调（Fine-tuning）采用率仅 9%，RLHF 仅 5%
- 2025 年推理大模型（DeepSeek-R1、QwQ-32B）结合 RAG 是期待方向

## 四大核心命题

### 命题一：数据价值的维度突破（检索什么）

- 问题：GIGO 定律制约，海量异构数据中识别核心信息、厘清关系网络
- 方案：构建分层知识图谱（Graph），从文本/语义检索到逻辑检索的维度突破
- 多层次知识库构建流程：Node（文档tree/工具/chunk）→ 问题（原子粒度）→ 社区（问题聚类）→ 主题（产品）→ 类目（产品线）
- 文档 Graph：切分提取文档/段落/Chunk，挖掘核心问题/摘要/因素，构建 Chunk 图谱
- 工单 Graph：根据工单间和工单内部 chunk 关系构图

### 命题二：异构检索的跃迁（怎么检索）

#### 问题理解
- 问题拆解：复杂问题按独立性原则拆解为子问题，简单问题不拆解
- 改写策略优化：报错信息保留英文原文、多产品引用支持、上下文感知改写
- 性能优化：两轮大模型改写压缩为一轮（4.5s→1.5s，降 60%+），高分知识命中旁路跳过改写
- 改写 GSB 评估：Good:Same:Bad = 61.51%:26.36%:12.13%

#### 检索策略
- 同义词库构建 + 分词器优化（aliws→ik_smart），recall@10 提升 4pt
- 混合检索架构：向量 + 文本 + 图谱三位一体
  - QQ 检索 + QA 检索 + 图谱召回
  - 图检索：关键节点向量和关键词检索，topN 节点子图返回
  - "小到大"策略获得更丰富上下文
- 迭代式检索：首次搜索无效时分层次递进查询（直接解决→部分解决→关联问题）

### 命题三：生成控制优化（怎么用检索内容）

#### 过滤无效参考信息（RAR）
- Retrieval-Augmented Relevance（RAR）：赋予大模型判断"不相关"的能力
- 核心方法：相关性数据 + 链式思维提示 + 动态少量示例学习 + 四级相关性定义
- Qwen 2.5 72B + Thoughts 表现最佳：错误召回率从 15.97% 降至 5.03%，错误减少 68.5%，F1=94.28%

#### 完善参考信息质量
- "small to big"策略：提供更长上下文
- 结构化 ID 过滤：每条参考信息分配唯一 ID
- 图片引用优化：Option 4（摘要+描述附加到参考信息末尾）出图比例最高

### 命题四：评估体系重构（RAG Diagnoser）

- 核心目标：细粒度诊断分析，从粗略判断到精准定位
- 流程：
  1. Query 分类（Factual-Y/N 等，参考 CoFE-RAG）
  2. 从历史工单提取 Ground Truth
  3. 原子事实提取（借鉴 RAG-Checker/RAGAS 的 claims 概念）
  4. 性能评估：原子事实准确率/召回率/矛盾率
- 模块评估：搜索诊断（各阶段覆盖率）、改写诊断（改写前后效果变化）、回复诊断（遗漏/错误原子事实）

## 技术背景数据

- 初期检索准确率 83%，生成准确率 66%（17pt 可信度断层）
- 优化后检索准确率 90%，生成准确率 85%

## 未来展望

1. 多模态检索：文本、图表、图像跨模态融合
2. DeepSearch：多层次异构图 + 长期思考推理融入搜索
3. 评估与优化：更科学全面的评估体系
