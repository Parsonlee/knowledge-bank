# 关于 Nano Banana 的一些浅思

> lencx 围绕 Nano Banana（Gemini 2.5 Flash Image）的使用经验与深层思考：从实用 prompt 模板到普通人借助 AI 进行创意表达的哲学探讨。

## Key Points

- **Nano Banana 定位**：Nano Banana 是 Gemini 2.5 Flash Image 的别名，原生多模态、快速高效，核心优势包括：
  - 原生多模态架构（图文统一处理）
  - 强角色一致性（跨图保持人物/风格稳定）
  - 内置世界知识（无需外部检索即可理解上下文）
  - 对话式编辑（自然语言迭代修改）

- **六大场景完整 Prompt 模板**
  1. **写实场景**：以摄影师思维描述——镜头焦距、光线方向、景深、被摄主体情绪与环境细节。
  2. **风格化插图**：以设计师思维描述——参考艺术流派/画家风格、色调体系、笔触质感。
  3. **图中文字**：明确指定字体风格、排版位置与文字内容（逐字准确）。
  4. **产品样机**：描述产品功能场景、材质光泽、背景留白与品牌调性。
  5. **极简/负空间**：强调留白比例、单一视觉重心、克制的色彩搭配。
  6. **连续分镜/故事板**：以帧序列描述动作变化、视角切换与情绪递进。

- **API 调用**：通过 Python `google-genai` 客户端调用，支持流式返回和多轮对话。

- **最佳实践**
  - 超具体描述（避免模糊词汇，用数值/专有名词量化细节）
  - 角色漂移修复（在后续 prompt 中重申关键外观特征）
  - 提供上下文与意图（说明图像用途，引导模型取舍）
  - 迭代精修（每轮只修改 1-2 个变量，观察差异）
  - 语义负向提示（描述"不想要"的元素而非单纯否定词）
  - 宽高比控制（明确指定横/竖/正方形及具体比例）
  - 摄影语言（使用光圈、快门速度等专业词汇引导视觉风格）

- **深层思考：普通人的 AI 创意表达**
  - 核心洞察：普通人无需掌握专业设计/摄影知识，只需学会"描述场景而非关键词"。
  - 传统创作需要工具技能（PS、摄影）+ 审美 + 想法；AI 时代只需清晰的想法表达能力。
  - 这重新定义了"创意门槛"——语言表达能力成为新的核心技能。

- **Gemini 的哲学回应（摘要）**：探讨人机协作中的认知谦逊——AI 应帮助人类从"验证者"转变为"探索者"，重新定义有价值的工作边界。

- **GPT-5 对 Gemini 回应的点评**：认为 Gemini 的表述过于乐观，真正挑战在于人类如何在 AI 辅助下保持主体性与批判性思维。

- **AI 编程（Vibe Coding）三阶段**
  1. **原型阶段**：快速生成，不在意代码质量。
  2. **审查-重构阶段**：人工介入，梳理结构、消除技术债。
  3. **迭代构建-审查循环**：功能增量开发，每轮人工验收。
  - 注意：UI/UX 不适合直接由 AI 生成，需人类主导设计决策。
