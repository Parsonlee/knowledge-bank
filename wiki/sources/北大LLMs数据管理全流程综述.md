## 核心要点

### 预训练阶段

#### 1. 数据规模
- 模型规模与训练数据集规模符合**缩放定律（Scaling Law）**：两者应以大致相同的速度扩大
- 随着模型规模增加，高质量训练数据可能耗尽；研究发现精心选择的重复数据可超越随机新数据

#### 2. 数据质量
- **去重**：减少模型记忆问题、避免训练-测试重叠；常用 N-gram 相似性与 MinHash 检测重复
- **质量过滤**：分类器、手工启发式算法或困惑度阈值过滤；注意过度过滤会导致性能下降（D4 论文）
- **毒性过滤**：启发式/规则/N-gram 分类器；过度过滤会降低泛化能力，增加有毒内容生成风险
- **社会偏见**：现有过滤器不足以解决，主要通过实验证明 LLM 包含大量偏见
- **数据多样性**：D2 剪枝——将数据集表示为包含困难分数的无向图，用正向反向消息传递选择多样性核心集

#### 3. 数据组合
- 高质量（如书籍）和高多样性（如网络）数据对模型性能有广泛帮助，即使与具体下游任务不完全相关
- 应囊括尽可能多的可靠数据来源，典型预训练集如 Pile：Common Crawl + Wikipedia + 书籍 + 医学/学术/代码/法律

#### 4. 数据管理系统
- **DataJuicer**：50+ 灵活数据管理操作符，支持零代码处理、低代码定制，适合及时反馈循环
- **Oasis**：交互式规则过滤 + 去偏神经质量过滤 + 自适应文档去重 + 整体数据评估

### SFT（有监督微调）阶段

#### 1. 数据规模
- 两种流派：①减少数据提高质量——LIMA 用 1000 个高质量样本取得不错效果；②扩大数据量
- 数据量影响因任务和模型规模而异：数学推理/代码生成随数据量稳步提升；一般能力约 1000 样本后提升缓慢

#### 2. 数据质量
- **指令质量**：用困惑度作为指标筛选候选指令（最重要因素）；用 LLM 为指令分配质量得分并迭代改进
- **指令多样性**：规模越大多样性越强性能越好；用 ChatGPT 作为细粒度标注器量化独特标签覆盖率
- **指令复杂度**：增加复杂性带来持续性能改进（非 token 数量增加导致）；WizardLM 等通过逐步重写增加复杂度
- **提示设计**：人类启发式设计 vs LLM 合成，两种方式优劣尚无定论

#### 3. 任务组合
- 多任务微调提高对未知任务泛化性；混合比例和任务平衡对有效指令微调至关重要
- 单任务专家 LLM 可以胜过多任务微调 LLM（LIMA 等研究支持）

### 挑战与未来方向
1. **需要更全面细致的理解**：质量与过滤效果权衡；去毒化与去偏见相互矛盾；少量高质量 vs 数据扩展
2. **通用数据管理框架**：需要更简单通用的系统帮助找到合适数据集
3. **幻觉问题**：预训练高质量语料 + SFT 阶段高质量指令数据，两条路径缓解幻觉
4. **社会偏见和公平性**：如何减轻预训练及 SFT 数据集中的潜在偏见
5. **多模态数据管理**：如何扩展/控制质量/平衡不同任务
