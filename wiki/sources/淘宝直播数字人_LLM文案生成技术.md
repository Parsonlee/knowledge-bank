## 核心要点

**业务背景**
- 数字人直播：话术经TTS转音频后驱动数字人形象，文案质量直接影响直播间呈现效果
- 旗舰版支持实时驱动，标准版/高级版在开播前离线生成AI话术存储于话术库
- 文案生成是"大脑"，与TTS强绑定，需考虑能否被正确念出

**一、基于语意的口播文案改写（解决TTS读错问题）**
- 问题：数字/英文/符号在不同语意下读法不同，无统一规则可处理（如88VIP读"八八VIP"不读"八十八"）
- 方案：LLM侧前置改写，直接输出TTS可用的纯净脚本
- 训练策略：
  - 难样本正负例挖掘+少量人工精标
  - DeepSeek-R1 数据增强扩充难样本
  - 基座 Qwen2.5-7B SFT + DPO（对比学习捕捉语意细微差异）
- 实验结论（4组对比）：
  - SFT人工精标基线 92% → R1通用增强 88%（通用增强无效）
  - → R1难样本增强 95% → +DPO 97%（针对性数据+对比学习双重增益）
- 关键发现：通用数据增强无法提升性能，长尾问题必须针对性干预

**二、去除机械感（口语化改写）**
- 问题：GPT直接生成文案书面感强，"口语化"指令模糊难follow
- 方案：依托 4000 小时真人直播 ASR 数据，提取语气词/停顿/表达方式，通过 DeepSeek-R1 蒸馏通用口语化改写模型
- 实验（3种训练策略）：
  - 无改写基线 46 → Qwen2.5-7B-Instruct SFT 79 → R1-Distill-Qwen-7B SFT 77
  - → R1-Distill-Qwen-7B SFT（含思考过程）84（思考过程显著提升口语化效果）
- 关键发现：直接生成改写 Qwen2.5-7B 优于 R1-Distill，但加入思考过程后 R1-Distill 反超

**三、多阶段多源信息文案生成（多步→单步蒸馏）**
- 多源数据：问评买信息（用户真实痛点）、实时权益信息（接口实时拉取，避免虚假宣传）、素材理解信息、个性化人设信息
- 人设经历库：RAG检索增强匹配合适人设经历片段生成个性化文案
- 训练数据生成：多步生成（GPT4o分步，fewshot增强follow能力）
- 推理部署：多步蒸馏到单步 Qwen2.5-7B（难follow指令用DPO正负例强化学习）
- 效果：单步蒸馏模型满意度 78.86 接近多步GPT4o的 80.03，优于单步GPT4o 75.53；生成耗时从 13.52s 降至 3.41s（缩减75%）

**四、素材文案结合（OCR+LLM图文一致）**
- 目标：商品图片实时替换数字人直播间背景，做到口播内容与背景图"图文一致"
- 方案：OCR提取图片文字 → 摘要抽取 → 图片分类（BERT，准确率87%）→ LLM生成对应讲解话术
- 素材理解难点：摘要错误、文本位置无序性、缺乏重点信息检测
- 过滤策略：kmeans聚类过滤+字数/字体规则过滤+坐标排序（上到下左到右）
- 模型架构：冻结 Qwen-0.5B + LoRA Adapter，Task1(OCR摘要) + Task2(图片选择) 任务隔离融合

**五、文案测评维度体系**
- 60分及格线（基础能力）：格式化、口语化、可信度、安全性、丰富度
- 80分良好：针对性卖点讲解（重点卖点/场景化描述/知识科普）、逼单话术购买欲
- 评估方法：规则匹配 + BERT相似度 + LLM打分三类方法
- 80分以上优秀文案暂无固定标准，后续以销冠真人主播文案为对标

**未来规划**
- 深入真人直播理解，结合 DeepSeek-R1 等推理模型提取更丰富真人讲解框架
- MLLM多模态理解视频/图片信息，从端到端多模态方向优化图文结合
- 仿照真人直播贴片实时露出权益信息
