# 概念：Coding 驱动 Agent（Code + 泛化调用）

**来源**：[[从代码生成到自主决策_Coding驱动的自我编程Agent]]

让 LLM 直接生成可执行 Python 代码来控制 Agent 行为，替代传统 JSON 函数调用方式，通过 Py4j 实现 Python→Java 的泛化调用。

## 核心思想

传统 Agent：LLM → JSON 参数 → 固定函数调用  
Coding 驱动：LLM → Python 代码 → 分支/循环/条件等任意控制流 → 工具执行

这让 Agent 拥有：
- **复杂控制逻辑**（分支、循环、异常处理），而非只能"下一步"
- **数据确定性**：Python 原生字符串/数字/时间运算保证精确计算
- **自我控制**：检查任务状态、查询自身技能

## FIM 代码续写格式

基于 Fill-In-Middle（FIM）技术，结构：
```
<|fim_prefix|> 前缀代码 <|fim_suffix|> 后缀代码 <|fim_middle|>（LLM 续写）
```
- 上下文感知：同时理解前后缀语义
- 精准填充：专门为中间代码片段生成优化
- 逻辑连贯：生成代码与前后文完美衔接

## 工具箱设计

每个工具箱（Toolkit）设计为 Python Class，工具为方法（函数），描述/属性映射为 Python 定义。LLM 以面向对象方式调用工具，代码可读性强。

**系统工具**（Agent 自控能力）：检查已发起任务、查询掌握技能、主动休眠、主动与用户对话、深度思考、解析数据。
