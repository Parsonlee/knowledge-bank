# 概念：激活值重计算

## 定义

一般会把前向计算的中间数据保存下来用于反向计算以避免反复计算。**重计算（Recomputation）**是指为降低显存消耗，先丢弃一些前向计算结果，在反向传播时再重新计算得到。属于"时间换空间"策略（消耗算力、带宽）。

## 激活值显存公式

结合论文《Reducing Activation Recomputation in Large Transformer Models》(arXiv:2205.05198)，不同并行/重计算组合下的激活值显存：

| 场景 | 公式 |
|------|------|
| 无重计算无并行 | `s*b*h*(34 + 5*a*s/h) * L * γ` |
| Tensor 并行（基线） | `s*b*h*(10 + 24/t + 5*a*s/h) * L * γ` |
| Tensor + 序列并行 | `s*b*h*(34 + 5*a*s/h)/t * L * γ` |
| 选择重计算 + Tensor 并行 | `s*b*h*(10 + 24/t) * L * γ` |
| 选择重计算 + Tensor+序列并行 | `s*b*h*(34/t) * L * γ` |
| 全部重计算 | `s*b*h*(2) * L * γ` |

参数说明：
- s 序列长度，b 微批量大小，h 隐藏维度，a 注意力头数
- t tensor 并行数值，L transformer 层数
- γ 精度系数（fp16 时为 1/(1024^3)）
