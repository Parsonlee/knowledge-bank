# 概念：DataOps 数据工程化

## 定义

将软件工程的 DevOps 理念应用于数据开发领域，实现数据工程的研发规范化、版本管理、测试、持续集成与部署，让数据开发像软件开发一样工程化。

## 核心思想

传统数据开发缺陷：
- SQL/Python 代码写在本地编辑器，复制粘贴到调度系统
- 无版本管理、无测试流程、无规范约束
- 难以复用，违反 DRY（Don't Repeat Yourself）原则

DataOps 目标：构建数据资产的 CMDB，像管理软件一样管理数据。

## 数仓建模的代码化

表格作为对象包含四类可代码化要素：
1. **基础信息**：表名、描述、所属业务域/主题域
2. **生产代码**：SQL / Python 逻辑
3. **Schema 定义**：字段名/类型/描述
4. **调度配置**：依赖/周期/优先级

打通发布流水线 → 实现数据开发的 CI/CD，并在事前/事中/事后插入检测约束。

## Python + SQL 结合

纯 SQL 开发的局限：
- 难以单元测试/单步调试
- 无流程控制（if/for）
- 代码复用性低

解决方案：Python 封装公共脚本（类似宏），实现类似软件工程的模块化复用。

## 与生产即治理的关系

DataOps 是实现[[概念_生产即治理]]的核心工程手段，通过流程约束而非事后扫描保证数据质量。
