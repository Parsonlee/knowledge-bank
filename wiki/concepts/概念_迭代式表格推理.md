# 概念_迭代式表格推理

> tags: RAG
> confidence: high

## 定义

迭代式表格推理是 TableRAG 提出的在线推理流程，应对需要跨多步、多种数据源（文本+表格）的复杂查询，将文本检索与 SQL 数据库操作结合的混合推理框架。

## 四个关键步骤

1. **上下文感知的查询拆解**：结合数据结构特征判断哪些子问题查表、哪些查文档（先从文本库找相关表格内容并链接到表结构描述，生成当前步骤子问题）
2. **文本检索**：向量召回 + 重排序，挑出 top-k 相关内容
3. **基于 SQL 的表格查询**：检查检索内容是否含表格数据，提取表结构，借助 NL2SQL（大模型驱动）生成 SQL 并在关系型数据库执行
4. **中间答案组合**：对比 SQL 结果（精确但可能出错）和文本片段（丰富但可能不完整）的一致性，根据可靠程度动态调整权重得出子答案

## 迭代终止与效率

- 搜集足够信息后迭代结束，输出整合答案
- 约 63% 问题在 5 步以内解决，30% 正好用 5 步

## 离线基础

依赖离线阶段构建的文本向量库（表格也序列化入库）和表格关系型数据库（保留原始结构），每个含表格的 chunk 与其"表格 schema"建立映射。
