# 概念：训练并行策略

## 背景

单卡物理显存基本不能满足大模型训练需求，需采用并行方式降低单卡显存消耗。这些方法出现在 [[实体_DeepSpeed]]、[[实体_Megatron]] 等并行框架中。

## 常见并行方法

- **TP（TensorParallel）**：tensor 并行
- **SP（SequenceParallel）**：序列并行
- **PP（PipelineParallel）**：pipeline 并行
- **DP（数据并行）**：主要用于提升 batch size，**不降低**单卡显存消耗
- **Zero**：参数服务器，分 Zero1/2/3，最早出现在 DeepSpeed 中（见 [[概念_Zero显存优化]]）
- **重计算**：见 [[概念_激活值重计算]]

## 并行后单卡显存公式

无并行时（单卡）：
`TotalMemory = Model + Optimizer + Activation + Gradient`

经并行策略调整后（举例 PP/TP/Zero1）：
`TotalMemory = Model/(PP*TP) + Optimizer/N + Activation/TP + Gradient/PP`

## 3D 并行

3D 并行主要指 TP(SP)/PP/DP。三者存在耦合关系：

`DP size = GPUs / (PP * TP)`

TP(SP)/PP 可降低模型、激活值、梯度的显存占用；DP 不降低单卡显存。

对显存计算的影响：
- 模型显存：`ModelMemory_3D ≈ ModelMemory / (PP * TP)`
- 梯度显存：`GradientMemory_3D ≈ GradientMemory / PP`

> 注意：梯度显存没有除以 TP，因为反向计算时需要 AllGather 出完整 gradient。

## MoE 影响

MoE 会改变模型参数分布进而改变计算，可视为多个小模型，改变的是模型结构（文中未展开具体计算）。

## 综合计算示例

条件：混合精度 + TP/SP/PP + 不开重计算 + Zero2

`TotalMemory = Model/(PP*TP) + (Optimizer + Gradient/PP)/N + Activation/TP`

其中：
- `Model = 2 * params / (1024^3)`
- `Optimizer = (4+4+4) * params / (1024^3)`
- `Activation = s*b*h*(34 + 5*a*s/h) * L / (1024^3)`
- `Gradient = 2 * params / (1024^3)`

> 注意：该公式只考虑模型部分，是估算值；实际还需考虑框架、分布式通信库、环境变量、算法副本数据。
