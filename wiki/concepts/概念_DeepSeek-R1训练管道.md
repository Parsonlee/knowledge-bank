# 概念：DeepSeek-R1 训练管道

DeepSeek-R1 的四阶段多管道训练流程，解决了 R1-Zero 可读性差、语言混杂等问题，同时进一步提升推理性能。

## 四阶段管道

### 阶段1：冷启动 SFT
- 收集数千条长推理链数据对 DeepSeek-V3-Base 做 SFT
- 输出格式：`|special_token|<reasoning_process>|special_token|<summary>`
- 目的：提供格式规范基础，避免纯 RL 早期不稳定

### 阶段2：面向推理的 RL
- 奖励 = 准确性奖励 + 语言一致性奖励（目标语言词汇占比）
- 专注数学/编程/科学/逻辑推理等密集推理任务
- 持续训练直到收敛

### 阶段3：拒绝采样 + SFT
- 从 RL checkpoint 拒绝采样，生成 60 万推理数据 + 20 万非推理数据（共 80 万）
- 推理数据：规则奖励/生成式奖励模型双路过滤，去除语言混杂/段落过长/代码块过多
- 非推理数据：写作/问答/翻译/自我认知，复用 DeepSeek-V3 SFT 数据集

### 阶段4：面向所有场景的 RL
- 推理任务：规则奖励（数学/编程/逻辑）
- 通用任务：奖励模型（人类偏好）
- 评估有用性：只看摘要部分；评估无害性：全输出含推理过程

## 奖励系统

- 只用规则奖励（准确性 + 格式），不用神经网络奖励模型，避免奖励黑客
- 格式奖励：要求推理过程在 `<think></think>` 标签内

## 失败尝试

- **PRM（过程奖励模型）**：中间步骤难以自动标注，易奖励作弊，无法大规模扩展
- **MCTS**：搜索空间远大于棋类游戏，易陷局部最优，模型训练困难
