# 概念：Agent 系统化工程

Agent 系统化工程是指将 AI Agent 从"能跑的 Demo"升级为"生产可靠系统"的工程方法论。

## 核心问题：LLM 不确定性放大

在 Agent 系统中，LLM 的不确定性会被多步调用显著放大：

- 假设单次交互正确率 90%
- 10 次交互 → 系统正确率 **35%**（0.9¹⁰）
- 20 次交互 → 系统正确率 **12%**（0.9²⁰）

三类放大维度：
1. **Memory 不确定性**：语义一致性问题，而非简单的存取问题
2. **编排动态性**：LLM 动态决定工具调用，决策空间爆炸
3. **测试不可预测性**：需回放测试/对比基线/模拟环境测试

## 三层复杂度模型

| 层次 | 状态 | 说明 |
|------|------|------|
| 可运行性 | ✅ 普遍支持 | 框架封装后开发门槛低 |
| 可复现性 | ⚙️ 局部支持 | 需自建状态与观测层 |
| 可进化性 | ❌ 仍靠人工 | 系统设计与工程实践 |

## 系统化方法论

**基本原则：**
- 先稳定，后聪明
- 先可观测，后优化

**核心功能：**
- 建立状态与日志可回放机制
- Prompt/Memory/RAG 版本追踪
- 引入成功率/漂移率/冗余调用等观测指标
- 明确每个 Agent 的边界与权限范围
- 预留"错误恢复"通道

**生产环境要求：**
- 像运维微服务一样设置监控/报警/故障恢复
- 安全边界：权限管理/越权调用/Prompt 注入防护
- 测试策略：回放测试+对比基线+模拟环境

## 认知演化路径（Level 0-4）

- Level 0：框架幻觉层（装框架跑 demo）
- Level 1：场景拼接层（RAG+工具原型）
- Level 2：系统设计层（可观测/状态管理）
- Level 3：工程落地层（SRE/安全/分布式视角）
- Level 4：智能演化层（长期记忆/自主学习，前沿）
