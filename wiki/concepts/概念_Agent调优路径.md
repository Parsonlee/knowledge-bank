# 概念：Agent 调优路径

阿里云服务领域总结的 Agent 构建与持续调优的渐进式路径，成本与效果随路径递增。

## 路径四级

### 第一级：提示词原型
- ROI 最低起点，直接用提示词工程构建
- 方法：AI 辅助生成（"最懂大模型的是大模型自己"）
- AI 可检测提示词中的冲突矛盾
- 验证：多次调优，观察运行稳定性

### 第二级：拆解 Workflow
- 当提示词调优仍不稳定时进入
- 构造更结构化的 Workflow 框架提高稳定性
- 代码规则编排可同时提升运行速度
- 三代演化：自然语言编排 → 代码/大模型混编 → 自然语言编排+LLM自主参考执行

### 第三级：Multi-Agent 架构
- 当 Workflow 固化/不灵活，或需要部分灵活+部分可控
- 复杂任务拆解为多个子任务，子 Agent 分别承接
- 设计子 Agent 间通信方式
- 端到端测试验证

### 第四级：模型定制化训练
- 前三级都无法满足时
- 构造领域训练数据（工具调用/多步/反问等语料）
- 执行 SFT/DPO/GRPO 等训练
- 评估：工具选择准确率 / 动作执行准确率 / 参数提取准确率

## Workflow vs LLM 自主规划选择原则

| 场景特征 | 推荐方案 |
|---------|---------|
| 标准化、重复执行、数学计算 | Workflow（避免幻觉） |
| 探索化、问题复杂、多种可能性 | LLM 自主规划 |
| 部分可控 + 部分灵活 | Multi-Agent（内稳外自主 or 内自主外稳） |
