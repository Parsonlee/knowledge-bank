# 概念_HTML代码方式生成视觉物料

> 用 LLM 生成 HTML/CSS 代码来渲染 PPT、封面等视觉物料的实现路径，来自 [[Gemini的PPT生成技巧与模板提示词]] 和 [[一套提示词实现封面自由]]。

## 定义

不直接生成图片，而是让 LLM 输出 HTML + CSS（必要时 + JavaScript）代码，在浏览器中渲染为 PPT 页面或封面图，再导出/截图为最终物料。

## 优势

- **可控性强**：相比直接出图，代码方式可以用提示词精细控制配色（精确到色值）、比例、字体层级、留白、装饰元素，所有细节都可调。
- **响应式 / 可复用**：用 Tailwind CSS、Google Fonts、Font Awesome 等在线 CDN 资源，保证跨浏览器一致比例，代码可直接运行。
- **可二次编辑**：输出的是结构化代码，可继续修改。

## 典型实现

- **Gemini PPT**：用前端代码方式实现，作者评价生成质量「比 Anthropic 那一坨强多了」，可导出 PDF / 谷歌幻灯片 / pptx。
- **封面生成提示词**（歸藏）：纯 HTML+CSS（公众号版用 Tailwind）+ html2canvas（小红书版）一键保存为图片，含图片下载按钮。

## 对模型前端能力的依赖

该路径强依赖 LLM 的前端代码能力。文章实测 DeepSeek V3 0324 前端能力「强的可怕」，可达到与 Claude 3.7 接近的封面生成效果。
