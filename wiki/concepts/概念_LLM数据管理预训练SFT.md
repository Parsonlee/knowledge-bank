# 概念：LLM 数据管理（预训练+SFT）

基于北大 "Data Management For Large Language Models: A Survey"（arXiv:2312.01700）。

## 预训练数据管理

### 数据规模
- **Scaling Law 约束**：模型大小和训练数据规模应以大致相同的速度扩大
- **重复数据利用**：精心选择的重复数据可超越随机新数据（D4 研究）

### 数据质量五要素
| 要素 | 方法 | 注意事项 |
|------|------|---------|
| 去重 | N-gram 相似性 + MinHash | 减少记忆/避免训练测试重叠 |
| 质量过滤 | 分类器/启发式/困惑度阈值 | 过度过滤导致性能下降 |
| 毒性过滤 | 规则/N-gram 分类器 | 过度过滤增加有毒生成风险 |
| 社会偏见 | 现有方法不足 | 去毒化与去偏见相互矛盾 |
| 数据多样性 | D2 剪枝（无向图+消息传递） | 高质量+高多样性来源最优 |

### 数据管理系统
- **DataJuicer**：50+ 操作符，零代码/低代码，适合及时反馈循环
- **Oasis**：规则过滤 + 神经质量过滤 + 自适应去重 + 整体评估

## SFT 数据管理

### 数据规模
- LIMA：1000 个高质量样本即可取得不错效果（"less is more"）
- 数学推理/代码生成：随数据量稳步提升；一般能力约 1000 样本后提升缓慢

### 数据质量四维度
1. **指令质量**：困惑度选择 + LLM 质量评分迭代改进（最重要因素）
2. **指令多样性**：ChatGPT 细粒度标注量化独特标签覆盖率
3. **指令复杂度**：逐步重写增加约束/推理/代码（WizardLM 方法）
4. **提示设计**：人类启发式 vs LLM 合成，优劣尚无定论

### 任务组合
- 多任务微调提高泛化性；但单任务专家 LLM 有时优于多任务微调

## 关键挑战

- 质量与过滤效果权衡（过滤阈值难以确定）
- 去毒化与去偏见相互矛盾
- 幻觉问题需从预训练和 SFT 两阶段共同应对
- 多模态数据管理（新兴方向）
