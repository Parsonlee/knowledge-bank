# 概念：A2A 协议（Agent to Agent）

**来源**：[[万字长文深入浅出教你优雅开发复杂AI_Agent]]

Google 联合 50+ 技术合作伙伴于 2025 年 4 月发布的开放协议，解决多 Agent 间互操作性与协作标准化问题。与 MCP 互补，MCP 面向工具上下文，A2A 面向 Agent 间通信。

## 核心功能

- **能力发现（Agent Card）**：智能体通过 JSON 格式"Agent Card"宣传自身能力，便于其他 Agent 发现和调用
- **任务和状态管理**：以任务为导向，定义任务对象及生命周期（分发/进度同步/结果工件输出）
- **协作消息交换**：Agent 间发送消息，交流上下文/回复/工件/用户指令
- **用户体验协商**：消息可包含图片/表单/视频等不同类型内容片段

## 与 MCP 的关系

**互补关系**（分层协同）：
- 单个 Agent 通过 MCP 协议调用工具
- 多个 Agent 通过 A2A 协议互相发现、分工协作

**竞争趋势**：工具可视为低自主性 Agent，Agent 可视为高自主性工具，两类协议边界逐渐模糊，未来可能进一步融合。

**选型建议**：
- MCP：交互模式类似传统 API 调用、输入输出明确、流程固定
- A2A：多 Agent 复杂协作、对话式交互、任务共同完成

## 先驱：ANP（2024 年 10 月，国人发起）

基于 W3C DID 身份认证，元协议层支持 Agent 自主协商沟通方式。目标打造智能体互联网 HTTP 协议，当前关注度较低（约 779 Star）。
