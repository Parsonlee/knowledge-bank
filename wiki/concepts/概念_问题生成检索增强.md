# 概念_问题生成检索增强

> tags: RAG, RAG/query
> confidence: high

## 定义

一种文本增强技术，通过 LLM 为每个文本片段生成相关问题，将问题与原始片段一起存入向量库。检索时用户查询既能匹配原始片段，也能匹配生成的问题，从而提高召回率和准确性。

## 流程

1. 文档切分为 text_document → 再切为 text_fragment
2. 对每个 fragment（或 document）调用 LLM 生成相关问题列表
3. 原始片段（ORIGINAL）与生成的问题（AUGMENTED）共同存入向量库
4. 问题的 metadata 指向其对应的原始文档
5. 检索时返回匹配到的原始文档作为上下文

## 生成级别

- **DOCUMENT_LEVEL**：对整个文档生成问题，问题覆盖面广
- **FRAGMENT_LEVEL**：对单个片段生成问题，问题更具针对性

## 关键设计

- Prompt 要求生成可从上下文直接回答的问题，不含答案或标题
- 去重过滤（`set()` + `clean_and_filter`）避免冗余
- 问题数量由 `QUESTIONS_PER_DOCUMENT` 控制
- 使用大模型 API 有成本考虑
