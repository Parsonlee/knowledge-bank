# 概念_LLM应用优化两轴

> tags: LLM, RAG
> confidence: high

## 定义

OpenAI DevDay 2023 提出的 LLM 应用优化思考框架：不沿线性路径(Prompt→RAG→Finetune)单向推进，而是沿两个正交轴并行迭代优化。

## 两个轴

### Context Optimization（上下文优化）
- 解决问题：模型缺乏相应知识（如私有数据）
- 核心手段：RAG（检索增强生成）
- 效果：注入新知识/更新知识、通过控制内容减少幻觉

### LLM Optimization（模型优化）
- 解决问题：模型不能产生正确输出（格式/风格/指令遵循）
- 核心手段：Fine-tuning
- 效果：激发已有知识、自定义结构/语气、教复杂指令

## 实践路线

1. Prompt Engineering 确定 baseline（最简单，不需额外组件）
2. 评估判断瓶颈是短期记忆（缺知识）还是长期记忆（缺能力）
3. 短期记忆不足 → RAG
4. 长期记忆不足 → Fine-tuning
5. 两者结合发挥各自优势

## 短期记忆 vs 长期记忆

- **短期记忆**：处理单一输入时的临时信息，模型缺乏特定信息来回答 → RAG 增强
- **长期记忆**：训练数据学到的知识，模型不能遵循特定结构/风格 → Fine-tuning 增强
