# 概念_多智能体协调

多个 LLM 智能体协同工作时的协调机制、常见挑战与工程原则。

## 核心挑战

- **协调复杂度指数增长**：智能体数量增加，交互模式急剧复杂
- **涌现行为**：主智能体 prompt 的微小变动会不可预测地影响子智能体行为
- **错误复合传播**：一步失败会导致整个智能体走向完全不同轨迹
- **状态管理**：长时间运行需跨多次工具调用维护状态，重启代价高
- **评测困难**：多智能体路径不确定，无法按"标准路径"评测

## 提示词工程原则（来自 Anthropic Research）

1. 像智能体一样思考——复现相同 prompt 和工具，逐步观察行为
2. 教会 orchestrator 如何分配任务——明确目标/输出格式/工具指引/任务边界
3. 按查询复杂度缩放努力——在 prompt 中嵌入缩放规则
4. 工具设计与选择同等重要——错误工具等于徒劳，清晰描述防止走错路径
5. 让智能体自我改进——Claude 4 可诊断失败并重写工具描述（任务时间减少 40%）
6. 先广后窄——短宽泛查询开始，再逐步收窄
7. 引导思考过程——extended thinking 规划 + interleaved thinking 评估
8. 并行工具调用——主智能体并行启动 3-5 子智能体，子智能体并行调用 3+ 工具

## 评测策略

- **小样本早期评测**：从 20 个查询开始，早期改动效果显著（30%→80%）
- **LLM-as-judge**：单次调用输出 0-1 分 + pass/fail，评估准确性/完整性/来源质量/工具效率
- **人工评测**：捕获幻觉、系统故障、来源选择偏差等 LLM 遗漏问题
- **端态评测**：评估最终状态是否正确，而非过程路径

## 生产可靠性

- 从错误发生点恢复（checkpoint + retry），不从头重启
- Rainbow deployment 渐进迁移流量
- 生产追踪监控决策模式（不监控具体对话内容）
- 上下文超限时 spawn 新子智能体，通过 Memory 传递计划
