# 概念：L0-L5 能力成熟度模型

## 定义

Hang Huang 提出的描述 AI 基础设施演进路径的模型，刻画 AI 从"工具使用者"逐级解锁到"系统主导者"的成长轨迹，类似游戏等级系统。

## 六个层级

- **L0 — Human-Oriented Legacy（实习生模式）**：模仿人类使用 CLI，靠"记忆"操作系统。被传统基础设施"卡脖子"——缺乏系统思维，只见树木不见森林（如只贴前端却不接后端服务依赖）。突破关键：基础设施从"人类语义"转向"机器语义"。

- **L1 — Tool-Driven Automation（工具熟练工）**：第一次拿到真正的"动手权限"，能通过标准化接口（MCP）调用工具改变系统状态，具备基础反馈循环。局限：把需求理解为"任务清单"而非"系统架构"，看不到工具间逻辑联系。

- **L2 — System-Level Modularity（系统理解者）**：主动从需求推导出系统架构，组合多个后端模块拼出可运行系统。需要：完整系统视图（系统状态 API 化）、模块组合自主权（结构化描述）、组合逻辑可编程。文章称这样的基础设施目前并不存在。

- **L3 — Runtime Programmability（动态架构师）**：获得"技术选型"自主权，自由搭配混合数据库/多运行环境/灵活部署，从 Dev 升级为 DevOps。现有 Railway/Fly.io 有 API 但非为 AI 工作方式设计。

- **L4 — Infra Synthesis（全栈掌控者）**：平台退居资源协调者，AI 掌握系统设计权/架构控制权/资源规划权。L3 控制运行，L4 主导建设（设计整个系统架构）。

- **L5 — Agent-Native OS（系统主人）**：控制边界彻底消失，AI 拿到 root 权限直接操作底层 OS（启 VM/装软件/配网络/挂 GPU/调内核），像人操作 Linux 一样自由。所有抽象层消失，真正进入 Agent-Native Computing 时代。
