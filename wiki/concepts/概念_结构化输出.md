# 概念_结构化输出

> tags: RAG
> confidence: high

## 定义

结构化输出（Structured Output, SO）是一种强制 LLM 以严格定义的格式（如 JSON Schema / Pydantic Schema）响应的方式。保证模型总是返回符合所提供 schema 的有效 JSON 数据。

## 实现方式

- 作为单独参数传递给 API（如 OpenAI response_format）
- Pydantic schema 中字段描述会被 LLM 视为 Prompt 的一部分
- 字段类型约束保证输出合规（如 `Union[float, int, Literal['N/A']]`）

## 与 CoT 结合（CoT SO）

RAG 挑战赛冠军方案中的 schema 设计：
- **step_by_step_analysis**：推理过程（CoT 本身）
- **reasoning_summary**：推理精简摘要
- **relevant_pages**：答案引用页码
- **final_answer**：简洁格式化答案

推理字段让模型充分思考后再输出结论，直接从 JSON 提取答案无需额外解析。

## SO 重解析器

- 不支持原生 SO 的模型：在 Prompt 中呈现 schema，大多数时候返回有效 JSON
- 验证失败时：用 `schema.model_validate(answer)` 检测，失败则重发 LLM
- 效果：合规率 100%，即使 8B 模型
