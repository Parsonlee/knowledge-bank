# 概念：显存组成与估算

## 定义

在模型训练/推理时，GPU 显存（全局内存）的消耗分为 AI 框架侧（用户可控）和系统层（驱动控制，用户不可控）两部分。

## 显存消耗内容

框架侧显存消耗包含以下内容：

| 类别 | 内容 | 可估算性 |
|------|------|----------|
| 可估算值 | 模型参数（parameter）、优化器状态值（optimizer_state）、激活值（activation）、梯度值（gradient）、输出数据（input） | 可通过公式计算 |
| 未命名数据 | 临时变量（temporary）、未知数据（unknown） | 不可估算 |
| 框架数据 | 自动梯度（autograd_detail） | 不可估算 |

## 各组件计算公式

### 模型参数（Model Memory）

`ModelMem = TypeSize * Params`

- fp32: `4 * params / (1024^3)` GB
- fp16/bf16: `2 * params / (1024^3)` GB
- fp8/int8: `1 * params / (1024^3)` GB

近似规则：1B 参数 fp32 ≈ 4GB（如 LLaMA-13B fp32 ≈ 52GB）。

### 优化器状态（Adam）

混合精度 Adam：`OptMem = (4+4+4) * Params / (1024^3)` GB
- 模型副本 4 Bytes + Momentum 4 Bytes + Variance 4 Bytes = 12 Bytes/参数

8位优化器：`8BitOptMem = (4+1+1) * Params / (1024^3)` GB

### 梯度值（Gradient）

与模型数据类型一致：
- fp32: `4 * params / (1024^3)` GB
- fp16/bf16: `2 * params / (1024^3)` GB

### 激活值（Activation）

参考 Megatron 论文公式：`s*b*h*(34 + 5*a*s/h) * L * γ` GB

参数：s=序列长度, b=微批量大小, h=隐藏维度, a=注意力头数, L=层数, γ=精度系数（fp16 时为 1/(1024^3)）。

## 估算误差

估算值与实际值可能相差 30% 以上，原因是未命名数据（temporary/unknown）占比过大。文中实例：估算 50GB 实际达到 75GB。

## 训练总显存（无并行）

`TotalMemory = Model + Optimizer + Activation + Gradient`

## 推理显存估算

`InferMemory ≈ 1.2 * ModelMemory`
