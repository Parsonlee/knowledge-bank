# 概念：Headless BI 指标中台

## 定义

Headless BI 是前后端分离的理念应用于数据分析领域：后端统一维护指标定义和计算逻辑，通过 SDK/API/类 SQL 向上层多个可视化应用提供服务，保障同一指标口径一致性。

## 核心问题

**传统模式**：同一指标在不同平台被多次重复定义 → 同名不同义/同义不同名，口径持续分裂。

## 架构设计

```
应用生态（报表/实验平台/目标管理）
        ↓ 统一 API / SDK / 类SQL
   ┌─────────────────────────────┐
   │  查询层（缓存/路由/转换）   │
   │  语义层（指标/维度元信息）  │
   │  数仓模型层（逻辑+物理表）  │
   │  物化加速层（Cube/OLAP/MPP）│
   └─────────────────────────────┘
```

## 指标标准化定义要素

- **度量**：视频总播放时长
- **维度**：性别、年龄、渠道...
- **业务限定**：体育类（过滤条件）
- **统计周期**：最近七天
- 自动生成计算 SQL

## 物化加速策略选择

| 场景 | 方案 | 优缺点 |
|------|------|--------|
| 常被观测、维度基数低、组合已知 | 预计算 Cube | 查询快、成本低；灵活性低 |
| 维度多、未来使用不确定 | OLAP 引擎（StarRocks/Clickhouse）| 任意维度；按高频维度建物化视图 |
| 指标定义时快速预览 | MPP 内存计算（Presto/Impala） | 低频操作，不需要持久化 |

## 与敏捷分析的张力

- 指标中台强调规范化，与"随时定义新指标"的敏捷分析存在矛盾
- 解决思路：在规范化基础上提高指标定义的**敏捷性**（规范不是枷锁，而是让"找到正确指标"变快）
