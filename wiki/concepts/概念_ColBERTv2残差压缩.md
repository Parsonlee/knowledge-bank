# 概念_ColBERTv2残差压缩

> tags: RAG, RAG/embedding
> confidence: high

## 定义

残差压缩（Residual Compression）是 ColBERTv2（SIGIR 2021）提出的存储优化技术，用于解决 ColBERT 多向量表示存储开销大的问题。核心是 k-means 聚类 + 残差量化。

## 改进动机

- ColBERT 为每个文档生成数百至数千个高维浮点向量
- 例：1000 token 文档、128 维 float32，单文档约 0.5MB；百万/十亿级文档存储压力巨大

## 残差压缩步骤

1. **向量聚类（找质心）**：对全库所有 token 向量做 k-means 聚类，生成 k 个质心，每个向量分配到最近质心（索引用少量字节存储）
2. **残差计算**：残差 = 原始向量 − 最近质心，因聚类使两者接近，残差值很小
3. **残差量化**：将残差每维从 float32（4 字节）映射到 uint8（1 字节，0-255）整数范围
4. **存储与重建**：存储 = 质心索引（2 字节）+ 量化残差（128 维×1 字节 = 128 字节）= 共 130 字节，对比原始 512 字节（128 维×4 字节）大幅减少；重建向量 = 质心 + 反量化残差

## 聚类为什么不可或缺

- 无质心则无法定义残差，压缩无从谈起
- 聚类让残差集中在小范围，量化误差可控
- 检索时只需检查与 query 靠近的质心组，计算量从 O(N) 降到 O(k)

## 效果

- 论文：存储减少 6-10 倍（24MB/文档 → 2-4MB），性能损失仅 1-2%
- BGE-M3 复现实验：8 文档，636KB → 159.16KB（约 4 倍压缩比），平均 MSE ≈ 0，相似度得分相对差异 <0.02%
