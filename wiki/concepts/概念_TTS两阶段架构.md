# TTS 两阶段架构

语音合成主流架构：语言模型（LM）阶段预测音频 token + 声学模型阶段将中间表征转为波形。

## 架构组成

### TTS 前端
- 文本归一化：缩写/数字/特殊字符处理
- 确定每个字的发音单元（中文用音素，其他语言用国际音标）

### TTS 后端（两阶段）
1. **语言模型阶段**：将文本/音素序列转为音频离散编码（中间表征）
   - VALL-E 之前：TTS 建模为回归问题，难以利用大量数据
   - VALL-E 之后：建模为离散信号分类问题，用 NLP 思路预测 token 采样概率
2. **声学模型 + 声码器**：中间表征→梅尔谱→波形

### 音频离散编码方法
- **Encodec**：语音领域 VQVAE，RVQ 量化残差，偏声学特征，易于重建
- **HuBERT**：语音领域 BERT，CNN encoder + transformer MLM，偏语义特征，易于生成

## 淘宝直播实践

- V1 使用 Encodec/HuBERT 编码 + 自回归生成
- V4 融合 [[实体_CosyVoice]] 2.0 架构：Qwen2.5-0.5B LLM 骨干 + 自研细粒度特征 + vLLM 加速
