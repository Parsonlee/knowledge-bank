# 概念：IFD（Instruction Following Difficulty，指令跟随难度）

## 定义

衡量模型对某条 SFT 数据的"必要性"指标，用于筛选对模型真正有训练价值的样本。

## 计算方式

```
s(A|Q) = (1/N) Σ log P(wᵢ | Q, w₁...wᵢ₋₁)   # 给定问题时回答的条件困惑度
s(A)   = (1/N) Σ log P(wᵢ | w₁...wᵢ₋₁)       # 单独回答的无条件困惑度
IFD    = s(A|Q) / s(A)
```

## 解读

| s(A\|Q) | s(A) | 含义 | 是否保留 |
|---------|------|------|---------|
| 高 | 低 | 模型难从 Q 生成 A，但 A 本身容易生成 → 指令跟随能力差 | **保留**（高 IFD） |
| 低 | 高 | 模型借助 Q 信息轻松生成 A → 已掌握该指令 | 不需要 |
| 高 | 高 | A 本身很难生成 → 可能数据有问题 | 不需要 |
| 低 | 低 | 模型对 Q 指令跟随很好 | 不需要 |

## 注意事项

- 脏数据（Q 与 A 关联差）也可能得高 IFD，存在误选风险；需配合质量过滤

## 使用流程（LIMA 风格）

1. 用 K-Means 聚类少量多样性数据（100 簇 × 10 条 = 1000 条）微调基础模型
2. 用该模型计算全量数据的 IFD
3. 选取 IFD 高的数据进行正式 SFT
