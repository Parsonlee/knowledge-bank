# 概念_向量数据库

> tags: RAG
> confidence: high

## 定义

向量数据库（Vector Database）是专门用于存储向量化表示的系统。通过向量索引和近似最近邻算法实现海量高维向量的快速相似性检索。与传统键值库（B-tree/Hash）不同，它能处理"含义相似"或"外观接近"的查询。

## 工作流程

1. **预处理**：通过 Embedding 模型将原始数据（文本/图像/音频）转为向量，归一化/降维/去重
2. **索引构建**：使用哈希/量化/图等技术建立索引结构（详见 [[概念_向量索引方法]]）
3. **相似度查询**：计算查询向量与索引中向量的相似度（详见 [[概念_向量相似度度量]]），返回 top-k
4. **后处理**：精确距离重排序、元数据过滤、多样性重排

## 大模型时代前的应用

- 人脸识别（亚毫秒延迟比对上亿人脸向量）
- 个性化推荐（Spotify 用 Annoy→Voyager/HNSW 做歌曲检索）
- 商品搜索（自然语言查询向量化后与商品向量匹配）
- 异常检测

## 大模型时代的驱动力

- LLM 训练数据有时效性限制，私有/实时知识需通过外部检索注入
- 上下文窗口限制：注意力矩阵 O(n²)，过长则慢/贵/遗忘
- RAG 将长文本压缩成向量，按需取用相关内容

## 主流产品

- [[实体_Faiss]]：Facebook 开源库，极致性能，非完整数据库
- Chroma：轻量 Python 向量数据库，LangChain 默认后端，适合原型开发
- Milvus：分布式向量数据库
- [[实体_Qdrant]]：Rust 编写的开源向量数据库
- Pinecone：云原生向量数据库服务
- Weaviate：支持 GraphQL 的向量搜索引擎
- [[实体_Elasticsearch]]：全文搜索+原生向量搜索，混合检索强项
