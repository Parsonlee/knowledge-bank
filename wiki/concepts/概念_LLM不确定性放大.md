# 概念：LLM 不确定性放大效应

在 Agent 系统中，单次 LLM 调用的不确定性，会随着调用链的延长被指数级放大。

## 数学模型

假设单次 LLM 交互正确率为 p：

- n 次交互后系统正确率 = pⁿ
- p=0.9，n=10 → **35%**
- p=0.9，n=20 → **12%**
- p=0.95，n=10 → **60%**

这意味着：**提升单次准确率比增加调用次数更重要**。

## 三个放大维度

### 1. Memory 不确定性放大

传统软件的状态管理是确定性的（数据库里有啥就是啥）。  
Agent 的 memory 依赖 LLM 解析/embedding/检索，结果高度不确定。  
本质：**memory 不是存取问题，而是语义一致性问题**。

### 2. 编排动态性放大

传统系统：编排是预定义固定流程。  
Agent 系统：LLM 动态决定调用哪个工具、如何调用。  
结果：决策空间爆炸，测试/监控/优化更复杂。

### 3. 测试不可预测性放大

传统软件：给定输入 → 预期输出（确定性）。  
Agent 系统：输出是概率分布，无法用传统单元测试保证稳定。  
需要：回放测试/对比基线测试/模拟环境测试。

## 工程应对策略

- **降低每步不确定性**：结构化输出、语法约束解码、低温度推理
- **缩短调用链**：合并步骤，减少 LLM 调用次数
- **增加校验节点**：执行后验证，幂等/重试/断路器
- **可观测性**：trace 每次调用输入输出，及时发现漂移
