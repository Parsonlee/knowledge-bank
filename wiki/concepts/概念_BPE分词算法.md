# 概念：BPE 分词算法（Byte-Pair Encoding）

## 定义

BPE 是一种子词（subword）级分词算法，通过迭代合并高频字节对，在词汇表大小与序列长度之间取得平衡，是 GPT 系列的核心分词方案。

## 核心思想

1. 从字节级别出发，统计所有相邻字节对的共现频率
2. 合并频率最高的字节对为新符号
3. 重复直到词汇表达到目标大小（如 GPT-4 约 100,277 个）
4. 得到可以还原原始文本的最小映射表（词汇表）

## 权衡

- **分词过细**（字符/字节级）：序列过长，计算开销大，注意力复杂度 O(n²) 爆炸
- **分词过粗**（单词级）：词汇表爆炸，内存占用高，OOV 问题严重
- BPE 在两者之间取得平衡，同时支持多语言、特殊字符

## 典型词汇表大小

| 模型 | 词汇表大小 |
|------|-----------|
| 原版 LLaMA | 32K |
| GPT-4 | 约 100K |
| 多语言模型（XLM-R/Bloom） | 约 250K |

## 实践影响

- 中文等非拉丁语言 token 效率低（需更多 token 表示相同语义）
- 不同 token 数影响推理延迟和上下文利用率
- 聊不同主题应单独开会话窗口，避免 token 浪费
