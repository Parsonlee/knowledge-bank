# 概念_上下文增强检索

> tags: RAG, RAG/retrieval
> confidence: high

## 定义

上下文增强检索（Context-Enriched Retrieval）是一种 RAG 检索策略：不只返回最相关的文本块，同时返回其前一个和后一个文本块，为 LLM 提供更完整的上下文信息。

## 核心思想

- 传统语义搜索只返回孤立的 top-k 文本块
- 孤立块缺乏上下文联系，可能导致 AI 信息不完整、回答错误
- 返回相邻块（context_window_size 控制窗口大小），帮助 AI 更好理解上下文

## 实现

- context_enriched_search(query, chunks, embeddings, top_k=1, context_window_size=1)
- 找到最相关块的索引 → start = max(0, idx - window) → end = min(len, idx + window + 1)
- 返回 [start, end) 范围内所有块

## 与相关方法的区别

| 方法 | 粒度 | 返回范围 |
|---|---|---|
| 标准检索 | 单块 | 仅最相关块 |
| 上下文增强检索 | 窗口 | 最相关块 ± N 块 |
| [[概念_父页面检索]] | 文档 | 完整父页面 |
| [[概念_RSE相关段落提取]] | 连续段落 | 价值最高的连续区间 |
