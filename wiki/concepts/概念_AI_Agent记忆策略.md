# 概念：AI Agent 记忆策略

AI 智能体的记忆策略是指在 LLM 上下文窗口有限的约束下，如何管理对话历史与长期信息的一组技术方案。

## 核心问题

LLM 存在上下文长度限制，长对话带来两个问题：
1. 遗忘早期信息，导致理解偏差
2. 过度消耗计算资源与成本

## 8 种主要策略

### 1. 全量记忆
所有历史无损保留，每次全量发送给 LLM。适合极短对话，不可持续。

### 2. 滑动窗口
固定大小队列，新进旧出，只保留最近 N 轮对话。实现最简单，但"健忘性强"。

### 3. 相关性过滤
为每条记忆分配重要性评分，容量满时删除最低分记忆。更智能，但评分机制难以精确。

### 4. 摘要/压缩
LLM 将早期对话压缩为摘要（Running Summary），与近期对话共同作为上下文。可与滑动窗口组合。

### 5. 向量数据库
嵌入存储 + 语义检索，突破上下文窗口限制，实现近乎无限的长期记忆。需要维护外部数据库服务。

### 6. 知识图谱
提取实体关系三元组，构建结构化知识图谱，支持多跳推理。可解释性强，构建维护成本高。

### 7. 分层记忆
短期工作记忆（滑动窗口）+ 长期向量库 + 提升机制，本质是多种策略的组合。适合跨天/跨月长期上下文感知。

### 8. 类OS内存管理
活动记忆（RAM/当前窗口）+ 被动记忆（Disk/外部存储），按 Page Fault 机制触发 page in。区别于分层记忆：超出窗口的全部换出，而非仅关键信息。

## 学术分类视角（翁荔框架）

- **感官记忆**：原始输入的嵌入表示
- **短期记忆（上下文内）**：Transformer 上下文窗口内
- **长期记忆（外部）**：跨会话可检索的向量存储（RAG）
- **参数化记忆**：知识直接编码进模型权重（微调/知识编辑/LoRA）
- **分层情境记忆**：情景记忆（具体事件）+ 语义记忆（抽象知识），通过反思机制提炼
