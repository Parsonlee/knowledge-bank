# 显存估算公式

## 显存消耗组成

大模型运行时显存由以下部分构成：

- 模型参数（parameter）
- 优化器状态值（optimizer_state）
- 激活值（activation）
- 梯度值（gradient）
- 输入数据（input）
- 临时变量（temporary）
- 自动梯度（autograd_detail）
- 未知数量（unknown）

## 各组件公式

### 模型参数
```
参数显存 = 参数量 × dtype_bytes
```
- FP32：4 字节/参数；FP16/BF16：2 字节/参数
- 例：7B 模型 FP16 ≈ 7e9 × 2 = 14 GB

### 梯度
```
梯度显存 = 参数量 × dtype_bytes
```
与参数同精度，FP16 下 2 字节/参数。

### 优化器状态（Adam，混合精度）
```
优化器显存 = 参数量 × 12   # FP32 主权重(4) + 一阶动量(4) + 二阶动量(4)
```

### 激活值
与 `batch_size × seq_len × hidden_size × num_layers` 成正比，是训练显存的主要变量。

### KV Cache（推理）
见 [[概念_KV_Cache|KV Cache]]：
```
KV Cache = 2 × batch_size × seq_len × num_layers × hidden_size × dtype_bytes
```

## 场景估算

### 推理显存
```
推理显存 ≈ 模型参数显存 + KV Cache + 临时变量
```
最低门槛约等于参数显存（如 7B FP16 ≈ 14GB + KV Cache）。

### 训练显存（混合精度 Adam）
```
训练显存 ≈ 参数量 × 16~20 字节 + 激活值
```
- 16 字节 = FP16 参数(2) + FP16 梯度(2) + 优化器(12)
- 激活值可通过 [[概念_激活重计算|激活重计算]] 降低

## 优化手段汇总

- [[概念_混合精度训练|混合精度训练]]：降低参数/梯度精度
- [[概念_激活重计算|激活重计算]]：用算力换激活显存
- [[概念_模型并行|模型并行]]：跨卡分摊参数与状态
- [[概念_KV_Cache|KV Cache]] 量化 / 分页：降低推理显存
