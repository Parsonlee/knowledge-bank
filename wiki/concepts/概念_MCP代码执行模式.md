## 定义

MCP 代码执行模式（Code Mode / CodeAgent 模式）是将 MCP Server 当作"代码 API"而非直接 Tool Call 调用的方式——LLM 通过**编写代码**来调用 MCP 工具，而非直接发出 TOOL CALL 请求。

由 Anthropic 工程团队提出，Cloudflare 独立发现并称之为"Code Mode"。

## 问题背景：Token 消耗两大瓶颈

随着 Agent 连接工具数量激增，出现两类严重的 Token 消耗问题：

1. **工具定义撑爆上下文窗口**
   - 数千个工具 = 几十万 Token 的工具描述
   - 20 个工具单次请求即可突破 60K Token
   - 根因：Host 将所有工具定义一次性注入上下文

2. **中间结果反复"过一遍"模型**
   - 示例：2 小时会议纪要被来回处理两次 = 额外 50,000 Token
   - 根因：每一步的中间结果都作为新的输入传给模型

## 核心解决方案

### 按需加载工具定义（文件树模式）

将所有 MCP Server 的工具生成**文件树**：

```
servers/
  google-drive/
    getDocument.ts
    listFiles.ts
    searchFiles.ts
  salesforce/
    createLead.ts
    updateContact.ts
```

Agent 通过**浏览文件系统**按需加载工具定义，而非一次性全量加载到上下文。

**效果**：Google Drive→Salesforce 示例，Token 从 150,000 锐减到 2,000，**节省 98.7%**。

### 两种工具发现方式

1. **文件系统浏览**：列出 `./servers/` 目录，按需读取所需工具的 `.ts` 定义文件
2. **Server 端搜索**：在 Server 上添加 `search_tools` 工具，支持按名称搜索，可选择信息详细程度

## 附加能力

### 隐私保护

代码执行环境中，中间结果默认只停留在执行环境内，**模型只看到 print/return 的内容**：

- 可在返回结果前先过滤（如 10,000 行电子表格只返回 5 行 pending 订单）
- Host 可自动对 PII 进行 Token 化脱敏
- 真实数据从 Google Sheets 流向 Salesforce 全程不经过模型

### 状态持久化

- 代码执行可将中间结果写入文件，支持**断点续作**
- Agent 可将有效代码沉淀为可复用**"技能"函数（Skills）**
- 配合 `SKILL.md` 描述文件逐步建立高级能力工具箱

## 代价与权衡

| 优势 | 代价 |
|------|------|
| Token 消耗减少 98.7% | 需要安全的代码执行环境（沙盒/资源限制/监控） |
| 隐私保护（数据不过模型） | 增加运营开销 |
| 状态持久化、断点续作 | 安全考量（代码注入风险） |
| 技能沉淀（Skills 复用） | 架构复杂度提升 |

## 与普通 MCP Tool Call 的对比

| 维度 | 普通 Tool Call | 代码执行模式 |
|------|--------------|------------|
| 工具加载 | 全量注入上下文 | 按需动态加载 |
| 中间结果 | 反复过模型 | 停留在执行环境 |
| Token 消耗 | 高（随工具数线性增长）| 极低（98.7% 节省） |
| 隐私保护 | 无（数据过模型）| 有（执行环境隔离） |
| 状态持久化 | 无 | 支持文件写入 |

## 适用场景

- 工具数量多（数十至数百个）的 Agent 系统
- 需要处理大量中间数据（大文件、大表格）的工作流
- 对数据隐私有要求的企业场景
- 需要技能沉淀与复用的长期 Agent
