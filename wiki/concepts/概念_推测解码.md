# 概念：推测解码（Speculative Decoding）

## 定义

**Draft-then-Verify** 范式：每个解码步先高效"推测"（Draft）target LLM 未来多步 token，再用 target LLM **并行验证**（Verify），通过验证的 token 作为解码结果。

## 动机

LLM 推理是 memory-bandwidth bound：大部分时间消耗在将参数从 HBM（High-Bandwidth Memory）搬到 cache，而非实际计算。自回归解码每步只生成 1 个 token，GPU 利用率极低。

## 关键性质

- **无损加速**：验证过程保证结果与 target LLM 自回归解码完全一致
- 支持 greedy decoding 和 nucleus sampling 的无损加速

## 关键要素

1. **推测（Drafting）策略**：高效性与准确性的权衡
   - Independent Drafting：同系列小模型推测（可通过知识蒸馏对齐行为）
   - Self-Drafting：Medusa/Blockwise Decoding 多 FFN Head；Early-Exiting；Layer-Skipping
2. **验证（Verification）策略**：保证质量
   - 贪婪验证：第一个不匹配 token 后全部丢弃
   - 放松验证：接受更多高质量推测 token（提高加速比）
   - 并行验证多条推测序列（Token Tree Verification）

## 代表工作

- Speculative Sampling（Google DeepMind，2022，arXiv 2211.17192）
- Accelerating LLM Decoding with Speculative Sampling（DeepMind，2302.01318）
- Medusa：多 FFN Head 并行预测
- SpecInfer：Token Tree Verification
