# 知识库配置清单

> 在任意一台桌面端按此配好后，`.obsidian/` 会随 Git 同步到其它设备，无需重复配置。
> （界面以中文版 Obsidian 为准；Obsidian Git 插件自身的设置面板是英文，已在下方括号标注中文含义。）

## 一、安装第三方插件

设置（右下角齿轮）→ **第三方插件** → 关闭「安全模式」→ 点「浏览」→ 搜索并安装 → 启用。

| 插件（搜索名） | 作用 | 优先级 |
| --- | --- | --- |
| **Obsidian Git** | 自动提交/推送/拉取，多端同步核心 | 必装 |
| **Templater** | 笔记模板（日记、会议、读书笔记等） | 高 |
| **Dataview** | 把笔记当数据库查询，自动生成列表/表格 | 高 |
| **Calendar** | 日历视图，配合日记 | 中 |
| **Tag Wrangler** | 批量重命名/管理标签 | 中 |
| **Outliner** | 列表大纲增强（拖拽、折叠） | 中 |
| **Paste image rename** | 粘贴图片时自动重命名归档 | 中 |

## 二、Obsidian Git 设置

设置 → 左侧最下方「第三方插件」分组里点 **Obsidian Git**（面板为英文）：

- **Auto pull on startup**（启动时自动拉取）→ 开 ✅
- **Vault backup interval (minutes)**（自动备份间隔/分钟）→ 填 `10`，即每 10 分钟自动提交并推送
- **Auto pull interval (minutes)**（自动拉取间隔/分钟）→ 填 `10`
- **Pull updates on every commit**（每次提交前先拉取）→ 开 ✅，减少冲突
- **Commit message**（提交信息）→ 可填 `vault backup: {{date}}`

## 三、文件与链接（设置 → 文件与链接）

- **新建笔记的存放位置** → 建议「仓库的根目录」或指定到 `notes/`
- **附件默认存放路径** → 建议「指定的文件夹」并填 `assets`，让图片集中归档
- **内部链接类型** → 保持「尽可能简短的形式」即可（跨平台不受影响）
- **使用 Wiki 链接** → 保持开启 ✅（即 `[[文件名]]` 双链）
- **始终更新内部链接** → 建议开启，重命名文件时自动修链接

## 四、附件归档

- 图片统一放 `assets/`（配合上面「附件默认存放路径」设置）。
- 装 **Paste image rename** 后，粘贴图片即自动命名，避免一堆 `Pasted image 2026...png`。

## 五、冲突处理

走 Git 多端，偶尔会有冲突文件（如 `note (conflicted copy).md`）。原则：
- 打开 Obsidian **先等它拉取完**再编辑。
- 真出现冲突时，用桌面端 `git status` / `git diff` 手动合并最稳。
